## Coverage gaps noted

- `test_archive_reports_behavior.sh` is the only bash test reaching into the
  archive_reports function. If that bash function is ever deleted, this is the
  last consumer to remove first. Worth confirming the Go-side
  `archive_reports_test.go` covers the silent-copy behavior the bash test
  documents.
- `test_classify_errors_dedup.sh`, `test_error_patterns.sh`, and `test_errors.sh`
  collectively duplicate Go-side coverage in internal/errors/classify_test.go.
  If Go coverage is verified complete, all three can go in a single sweep.
- The m17 shim's `classify_routing_decision` token vocabulary is only verified
  by `test_error_patterns_classify_threshold.sh` (KEPT). Recommend mirroring
  that contract assertion into a Go test in internal/errors so the shim can
  eventually be deleted without losing the contract.
- `test_gates_stale_raw_errors.sh` (NEEDS-REVIEW) sits on the m17 wedge boundary
  — confirm whether `tekhton diagnose classify --mode all` cleans/reseeds
  BUILD_RAW_ERRORS.txt itself or relies on the bash gate to do so. The bash
  test asserts the gate does the cleanup.
