## TUI Sidecar Orphan Fix and Stage Timings Polish (v3.125.4)

**One intent. Many hands.**

This patch closes a TUI sidecar orphan that left `tools/tui.py` running
indefinitely after a build-gate-failure exit, and tightens the Stage Timings
panel so long substage breadcrumbs no longer push the time/turns columns
off-screen.

## Highlights

- Fixed TUI sidecar orphan after build-gate-failure exit. When the build gate
  failed and the parent shell exited via `error "…"; exit 1` (e.g.
  `stages/coder.sh:1167` and sibling sites), the sidecar could remain alive
  indefinitely with a stale `.claude/tui_sidecar.pid` left on disk. Three
  coupled fixes:
  - `lib/tui.sh::tui_stop` — dropped the `_TUI_ACTIVE` early-return guard and
    added a pidfile fallback so the EXIT trap reaps the sidecar even when
    `_TUI_ACTIVE` was flipped false earlier in the run. Pidfile is now removed
    on every code path (alive process, dead PID, or no PID at all).
    `tui_complete` keeps its happy-path guard — the asymmetry is intentional.
  - `tools/tui.py` — added a double-timeout watchdog escape hatch that fires
    on `2 × watchdog_secs` of status-file staleness regardless of
    `current_agent_status` / `agent_turns_used`, closing the gap where the
    original watchdog preconditions were unreachable in the orphan scenario.
  - Regression coverage: `tests/test_tui_stop_orphan_recovery.sh`,
    `tests/test_tui_orphan_lifecycle_integration.sh`, and watchdog tests in
    `tools/tests/test_tui.py`.
- Polished the Stage Timings panel with a 32-char cap on stage labels.
  Substage breadcrumbs like `wrap-up » running final static analyzer` were
  pushing the right-aligned time/turns columns off-screen. `_truncate(s, limit)`
  was promoted from `tools/tui_render.py` into a new `tools/tui_render_common.py`
  module (avoids a circular import) and applied to both completed-stage labels
  and the live-row breadcrumb in `tools/tui_render_timings.py`. The column's
  existing `overflow="fold"` remains as a backstop on narrow terminals.
- Documented a 300-line file ceiling exemption for data-only files. Added an
  explicit carve-out in `CLAUDE.md` (Non-Negotiable Rule #8) for files
  containing only `:=` defaults, constant declarations, and shared
  clamp/validation helper calls — no function bodies, no conditional logic.
  `lib/config_defaults.sh` is the canonical example.

## Why it matters

- **Operator trust in the TUI lifecycle.** The orphan sidecar produced a
  visibly hung terminal even after the parent pipeline had cleanly exited, and
  the stale pidfile interfered with the next run's startup. The combined
  pidfile fallback + double-timeout watchdog mean the sidecar is now
  unconditionally reaped by the EXIT trap on every error-exit path, with a
  staleness-only watchdog as the safety net.
- **Readable timings on real terminals.** Substage breadcrumbs were a
  legitimate readability regression on standard 80–120 column terminals. A
  single-source `_truncate` helper keeps timing rows aligned without losing
  the column overflow backstop.
- **Clearer convention for declarative config files.** Recognising
  `config_defaults.sh` as a data file rather than a code file removes a
  recurring false-positive against the 300-line ceiling.

## Upgrade notes

No configuration or API changes. Users of the brew tap will receive this patch
automatically on next `brew upgrade tekhton`.
