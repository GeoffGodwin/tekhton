## Release Workflow and GitHub Pages Hotfixes (v3.125.3)

**One intent. Many hands.**

This patch fixes three regressions that caused the v3.125.2 release workflow to
emit spurious failure emails and produce a broken GitHub Release.

## Highlights

- Fixed `gh release create` failure caused by unsupported `--latest` flag in
  the installed `gh` CLI version; GitHub marks the highest semver tag as latest
  automatically.
- Fixed Jekyll build failure on GitHub Pages caused by an unguarded `{%s}`
  printf format string in a `DESIGN_v4.md` code block being misread as an
  unterminated Liquid tag; block is now wrapped in `{% raw %}...{% endraw %}`.
- Fixed missing `{% endraw %}` closing tag that would have caused the raw block
  to swallow the remainder of the document.

## Why it matters

- The `gh release create` error caused the release script to exit with a
  warning even though the tag was already pushed, leaving the GitHub Release
  page unpopulated until created manually.
- The Jekyll Liquid error blocked GitHub Pages from rebuilding, so the live
  docs site remained pinned at the previous version despite a successful
  deployment run completing.

## Upgrade notes

No configuration or API changes. Users of the brew tap will receive this patch
automatically on next `brew upgrade tekhton`.
