## Context-Aware Pipeline (V3 feature complete)

**One intent. Many hands.**

V3 is feature complete. **66 milestones** delivered across 10 themes, making the pipeline **context-aware** — milestones live as a dependency graph with a sliding context window, agents receive ranked file signatures relevant to their task, and a suite of new stages and modes make the pipeline safer, faster, and easier to use.

## Highlights

### Context & Indexing (M01–M08)
- **Milestone DAG** — file-based milestones with `MANIFEST.cfg` dependency tracking, sliding context window, parallel groups, automatic migration from inline CLAUDE.md milestones
- **Tree-sitter repo maps** with PageRank ranking, task-relevant context slicing per stage, cross-run file association tracking, repo map cross-stage cache
- **Serena LSP integration** via MCP for live symbol lookup
- **Prompt tool awareness** so agents actually use the indexing tools available to them

### Quality & Safety Stages
- **Security agent** (M09) — dedicated OWASP-aware review stage with severity scoring and auto-remediation
- **Task intake / PM agent** (M10) — clarity scoring, scope assessment, decomposition
- **UI/UX specialist** (M59) — auto-enabled on UI projects, 8-category checklist (component structure, design system, WCAG 2.1 AA, responsive behavior, state presentation, interaction patterns)
- **Test baseline & hygiene** (M63) — pre-existing failure detection, completion gate hardening
- **Tester surgical fix mode** (M64) — scoped fix agents on test failures
- **Build gate hardening** (M30) — hang prevention, timeout enforcement, process tree cleanup

### Environment Intelligence (M53–M56)
- Error pattern registry with build gate classification
- Auto-remediation engine for cataloged failure patterns
- Pre-flight environment validation with optional auto-fix
- Service readiness probing & enhanced diagnosis

### UI/UX Design Intelligence (M57–M60)
- Platform adapter framework — file-based adapters in `platforms/`
- Web UI: Tailwind, MUI, shadcn, Chakra, Bootstrap detection
- Mobile & game: Flutter, SwiftUI/UIKit, Jetpack Compose, Phaser, PixiJS, Three.js, Babylon.js
- User-extensible via `.claude/platforms/<name>/`

### Watchtower Dashboard (M13–M14, M34–M38, M66)
- Browser-based monitoring with Live Run, Milestone Map, Reports, Trends, Security Summary tabs
- Smart refresh, action items severity colors, interactive controls
- Full-stage metrics with hierarchical breakdown
- Parallel teams readiness (V4 prep)

### Notes Pipeline Rewrite (M33, M39–M42)
- Core rewrite with triage & sizing gate
- Tag-specialized execution paths (BUG/FEAT/POLISH)
- Notes injection hygiene + action items UX
- `note` subcommand for managing HUMAN_NOTES.md from the CLI

### Developer Experience (M17–M27, M52)
- **Express mode** — zero-config execution when no `pipeline.conf` exists
- **TDD support** — `PIPELINE_ORDER=test_first`
- **Browser planning** — interactive `--plan-browser` with answer persistence
- **Dry-run preview** — `--dry-run` runs scout + intake only
- **Rollback** — `--rollback` reverts the last pipeline run
- **Pipeline diagnostics** — `--diagnose` with structured recovery suggestions
- **Version migration framework** — `--migrate`
- **Onboarding flow fix** — repaired the circular `--init` ↔ `--plan` handoff

### Acceleration & Telemetry (M43–M50)
- Test-aware coding with jr coder test-fix gate
- Scout prompt leverages repo map & Serena
- Instrumentation & timing report (per-stage, per-agent)
- Intra-run context cache to avoid redundant file reads
- Reduced unnecessary agent invocations via smarter skip logic
- Structured run memory (JSONL) for cross-run learning
- Progress transparency with timing estimates from run history
- Causal event log (JSONL) for structured debugging

### Brownfield Intelligence (M11–M12, M15, M22)
- AI artifact detection with archive/tidy/ignore handling
- Workspace, service, CI/CD, infrastructure detection
- Documentation quality assessment, test framework detection
- Project health scoring with five-category assessment and belt ratings

## Get started

```bash
git clone https://github.com/geoffgodwin/tekhton.git
cd tekhton && chmod +x tekhton.sh
cd /path/to/your/project
/path/to/tekhton/tekhton.sh --init
```

See `README.md` for full quick-start, `DESIGN_v3.md` for the design document with retrospective, and `MILESTONE_ARCHIVE.md` for individual milestone records.
