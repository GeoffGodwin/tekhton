## Genesis (pre-semver)

The genesis of Tekhton, retroactively tagged.

This release represents the "v1 era" — Tekhton's earliest period, when it was extracted as a standalone multi-agent development pipeline tool from a parent project. During this phase `TEKHTON_VERSION` was the literal string `"v1"` with no minor or patch component. Formal semver and milestone numbering were both introduced in the v2 era.

Tagged at PR #13 (`Feature/security pass`) — the last v1 PR on `main` immediately before the V2 integration merge (PR #47). This matches the **"1.0 Baseline"** architecture described in `DESIGN_v2.md`: a five-layer pipeline (entry point + stages + libraries + prompt templates + agent roles) with FIFO-isolated agent invocation, drift observation, and the planning phase.

## Highlights (PRs #1–#13)

- Initial extraction of the multi-agent pipeline as a standalone tool
- Architecture-aware prompts with ACP (Architecture Change Proposals) and drift observation
- Drift log infrastructure
- Architect audit agent with remediation routing
- Dependency constraint validation in the build gate
- Agent exit detection and `run_final_checks` safety
- Interactive planning phase — DESIGN.md + CLAUDE.md generation (PR #8)
- Planning phase enhancements (PR #10) and follow-up improvements (PR #11)
- Man page, usage, and version flag implementation (PR #12)
- Security pass — sandboxing and credential redaction (PR #13)

## Note on numbering

This release is tagged retroactively for historical record. The actual project shipped without an explicit v1.0.0 release; it transitioned directly into v2 once semver was adopted and the V2 work on the `feature/Version2` branch was merged into `main`.
