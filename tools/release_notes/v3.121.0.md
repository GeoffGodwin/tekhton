## Planning Path Hardening and Reliability (V3.121.0)

**One intent. Many hands.**

V3.121.0 ships two tightly-coupled milestones (M120 and M121) focused on
planning-mode correctness, safer defaults, and fail-loud behavior when design
file paths are invalid.

## Highlights

### DESIGN_FILE default restoration and init safety (M120)

- Fixed a planning-path regression where `--init` could emit
  `DESIGN_FILE=""` into `pipeline.conf`, poisoning subsequent planning runs.
- Restored safe defaulting so planning entrypoints recover to the canonical
  design path when legacy or empty config values are encountered.
- Preserved user overrides when a non-empty custom `DESIGN_FILE` is set.
- Added clearer end-of-init next-step guidance that distinguishes greenfield
  and brownfield project posture.

### Write-failure hardening and test coverage (M121)

- Added shared guardrails for planning/replanning flows so empty or directory-
  shaped `DESIGN_FILE` values fail loudly instead of producing ambiguous state.
- Hardened design-document write behavior with explicit failure checks and
  post-write verification to prevent silent success on invalid writes.
- Expanded validation feedback for malformed `DESIGN_FILE` values (empty string
  and trailing slash directory shape).
- Added dedicated tests for:
  - Empty-slate `--init` -> `--plan` roundtrip
  - Plan-config loader behavior across empty/custom/unset `DESIGN_FILE`
  - Regression protection for previously silent planning-path failures

## Why this release matters

This release improves planning reliability in exactly the paths new and existing
users hit first: initialization, plan generation, and replan flows. The main
outcome is stronger correctness guarantees with clearer failure modes and more
robust CI coverage for configuration edge cases.
