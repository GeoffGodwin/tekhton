## Adaptive Pipeline 2.0 (final)

The "v2 era" of Tekhton — the **Adaptive Pipeline 2.0** initiative, developed on the `feature/Version2` branch and merged into `main` on March 21, 2026 via PR #47. Tagged at that integration merge — the canonical "V2 final" state on `main`.

V2 delivered **21 milestones**, introduced formal `MAJOR.MINOR.PATCH` semver (where MINOR = milestone number), and established the architectural foundation that V3 would build the Milestone DAG and intelligent indexing on top of. See `DESIGN_v2.md` for the full design document.

## Themes Delivered (M1–M21)

### Adaptive Pipeline 2.0
- Token & context accounting with measured budgets
- Context compiler — task-scoped artifact assembly
- Milestone state machine and auto-advance
- Mid-run clarification and replanning protocol
- Specialist reviewers framework (security, performance, API contracts)
- Workflow learning and run metrics with adaptive calibration
- Autonomous debt sweeps via the `cleanup` stage

### Brownfield Intelligence (Smart Init)
- Project crawler & index generator
- Tech stack detection engine
- Smart init orchestrator
- Agent-assisted DESIGN.md / CLAUDE.md synthesis
- Incremental rescan & index maintenance via `--rescan`
- Brownfield replan via `--replan`

### Autonomous Operation
- `--complete` loop with safety bounds
- `--auto-advance` milestone chaining
- `--human` mode — process HUMAN_NOTES.md items as tasks
- Transient error retry with exponential backoff
- Turn-exhaustion continuation
- Pre-flight milestone sizing & null-run auto-split
- Outer orchestration loop

### Quality & Safety
- Error taxonomy, classification engine, sensitive data redaction
- Agent output monitoring & null-run detection
- Observability & error attribution
- Security hardening — anti-prompt-injection, sanitized config parsing, agent permission scoping
- Milestone commit signatures and completion signaling
- Init UX overhaul

## Integration

This merge brought ~60K lines across ~980 files into `main` as a single integration of the `feature/Version2` branch. After this commit, V3 work began (PR #49 onwards introduced what became V3M1; PR #51 formalized the V2-to-V3 milestone format conversion).
