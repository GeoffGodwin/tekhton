## TUI Lifecycle Hardening and Invariants (V3.119.0)

**One intent. Many hands.**

V3.119.0 is a stability-focused release that consolidates the recent TUI work
from M104 through M119. The core theme is lifecycle correctness: stage state,
substage state, event attribution, and completion timing now stay coherent
through normal runs, skips, rework loops, and finalization.

## Highlights

### Lifecycle and Substage Model (M104-M116)

- Introduced operation-level liveness wrapping via `run_op`, then migrated it
  onto the hierarchical substage API.
- Added the M113 substage model and moved scout/rework/architect-remediation
  flows to the same protocol surface.
- Ensured TUI timing and stage semantics remain consistent as ownership changes
  between parent stages and active substages.

### Event Fidelity and Stage Wiring (M107-M118)

- Wired all pipeline stages into the protocol-driven TUI stage lifecycle.
- Added event `source` attribution so Recent Events reliably identify where each
  message originated.
- Closed timing/ordering gaps where stage success messaging could appear before
  pill completion state was fully finalized.

### Quality Gate and Documentation (M119)

- Added lifecycle-invariant documentation and validation coverage as a dedicated
  quality gate milestone.
- Improved guardrails so lifecycle regressions are easier to detect during test
  runs and release validation.

## Why this release matters

This release does not primarily add new user-facing pipeline stages. Instead, it
raises confidence in the runtime behavior that powers existing workflows:
clearer stage ownership, cleaner event provenance, and stronger correctness
checks around TUI completion semantics.
