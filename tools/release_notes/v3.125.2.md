## Indexer Hardening and Quota-Pause Reliability (V3.125.2)

**One intent. Many hands.**

This release rolls up improvements shipped since `v3.121.0`, with a focus on
indexer reliability and quota-pause resilience.

## Highlights

- Fixed TypeScript/TSX tree-sitter multi-grammar loading so TS-heavy projects
  no longer silently lose repo-map generation.
- Added stronger indexer diagnostics, including startup grammar audit and
  clearer fallback stderr surfacing.
- Improved pipeline quality flow by moving acceptance-criteria linting to the
  intended earlier gate.
- Fixed milestone UI state leakage in auto-advance runs for cleaner TUI state
  transitions.
- Fixed CI workflow checkout regressions affecting release/pages reliability.
- Added TUI-aware quota-pause behavior with clearer pause visibility,
  countdowns, and spinner coordination.
- Added Retry-After-aware scheduling, layered probe strategy, and backoff/jitter
  controls for more accurate and lower-cost quota recovery.
- Closed out final non-blocking cleanup for this release train.

## Why it matters

This release train strengthens two high-impact reliability surfaces:

- **Repo intelligence reliability:** indexer behavior is now safer for
  TypeScript-heavy codebases, with earlier and more actionable diagnostics.
- **Long-running pipeline resilience:** quota pauses are now observable,
  schedulable, and better aligned with real upstream refresh semantics.

The result is fewer silent degradations and clearer operator feedback during
both startup and long-running autonomous runs.
