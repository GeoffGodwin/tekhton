## Non-Blocking Findings (MEDIUM/LOW)
- [LOW] [category:A03] [adapter.go:369-380] fixable:yes — `buildBashScript` uses Go `%q` to double-quote `tekhtonHome`, `projectDir`, and `scriptPath` in the generated bash script. Go's `%q` does not escape bash metacharacters (`$`, backticks); a path such as `/tmp/$(id)` would trigger command substitution when bash evaluates the script. In production these values come from trusted CLI flags (`--tekhton-home`, `--project-dir`), but if either field is ever sourced from a config file or other partially-trusted input the exposure escalates. Fix: switch to single-quoted bash strings with `'` → `'\''` escaping (immune to `$` and backtick expansion).
- [LOW] [category:A09] [adapter.go:299] fixable:yes — Log files are created with mode 0644 (world-readable). Stage output may include environment variables, API tokens, or other secrets emitted by the bash subprocesses. Fix: use mode 0600 so log files are readable only by the process owner.
- [LOW] [category:A03] [adapter.go:267] fixable:yes — `EnvOverrides` key validation only rejects empty strings; keys containing `=` (e.g. `"FOO=BAR"`) produce the env entry `FOO=BAR=value`. Linux `execve` splits on the first `=`, so the actual key is `FOO` with value `BAR=value`, but the parent-env deduplication loop (lines 248-252) compares against the literal map key `"FOO=BAR"` and therefore does not drop the parent's `FOO` entry — both survive, with undefined precedence. This is an internal API concern (no user-controlled path to `EnvOverrides` today), but should be guarded: reject keys containing `=` or `\x00`.
