## Non-Blocking Findings (MEDIUM/LOW)
- [MEDIUM] [category:A08] [.github/workflows/brew-bump.yml:30] fixable:yes — GHA script injection: `${{ steps.sha.outputs.tag }}` is interpolated inline into a `sed` run command rather than passed via an `env:` block. Git tag names cannot contain `;`, `&`, or most shell metacharacters, so practical exploitation is constrained, but the pattern violates GitHub's own hardening guidance and could corrupt the formula if a tag containing a sed delimiter (e.g. `|`) were ever pushed. Fix: move the value into a step-level `env:` variable (e.g. `TAG: ${{ steps.sha.outputs.tag }}`) and reference it as `$TAG` in the shell command.
- [MEDIUM] [category:A06] [.github/workflows/brew-bump.yml:20] fixable:yes — Unpinned action: `actions/checkout@v4` references a mutable major-version tag rather than a pinned commit SHA. If the action is compromised or updated maliciously, the `HOMEBREW_TAP_PAT` secret could be exfiltrated or the tap repo pushed to arbitrarily. Fix: pin to a full commit SHA (e.g. `actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683` for v4.2.2).
- [MEDIUM] [category:A05] [.github/workflows/brew-bump.yml] fixable:yes — No `permissions` block declared at the workflow or job level. The workflow inherits the repository's default token permissions (typically broad read/write on contents). The bump job needs no repository access at all (it checks out the tap via PAT, not the default token); the smoke-test job needs only `contents: read`. Declare `permissions: contents: read` at the top level to enforce least privilege.
- [LOW] [category:A05] [docs/getting-started/installation.md:22] fixable:yes — The recommended curl|bash one-liner fetches `install.sh` from the mutable `main` branch ref. If `main` is compromised between releases, users running this command would execute the modified script without any integrity signal. Consider fetching from a versioned tag URL and documenting the expected sha256 alongside the review step already present in the docs.
