## Architecture Decisions

- **Context reporting is per-stage, not per-agent**: Each stage (coder, reviewer, tester) reports its own context breakdown. This matches the architecture where stages own context assembly — the context library just measures what stages give it.
- **Accumulator pattern for components**: `_add_context_component()` accumulates into module globals, then `log_context_report()` finalizes and resets. This avoids passing arrays through bash and keeps the stage integration clean (one call per block, one finalize call).
- **LAST_CONTEXT_TOKENS/LAST_CONTEXT_PCT exported for summary**: The run summary reads these globals after the agent completes. This is the same pattern used by LAST_AGENT_TURNS/LAST_AGENT_EXIT_CODE.
- **No behavioral changes**: This milestone is measurement-only. Over-budget warnings are logged but do not block or modify agent invocations. Milestone 2 (Context Compiler) will add compression when over budget.
