# Scout Report — Fix Two Bugs in HUMAN_NOTES

## Relevant Files

- **lib/notes.sh** — Contains `resolve_human_notes()` function (lines 89-187). The fallback logic (lines 170-186) unconditionally resets [~] → [ ] when CODER_SUMMARY.md is missing, even on successful pipeline completion.
- **lib/milestone_ops.sh** — Contains `clear_milestone_state()` function (lines 278-285). Removes the MILESTONE_STATE_FILE but is never called from anywhere.
- **stages/coder.sh** — Calls `resolve_human_notes()` at three locations (lines 389, 434, 442) after coder runs. Currently passes no parameters.
- **tekhton.sh** — Post-pipeline finalization block (lines 960-1165). Calls `archive_completed_milestone()` twice (lines 1137, 1154) after successful commits but never calls `clear_milestone_state()`.
- **lib/milestone_archival.sh** — `archive_completed_milestone()` function (sourced by tekhton.sh). Called after successful milestone completion and commit.

## Suspected Root Cause Areas

**Bug 1 — resolve_human_notes() missing exit-code awareness:**
- `lib/notes.sh` lines 170-186: Fallback path unconditionally resets [~] → [ ] when CODER_SUMMARY.md is absent. Does not check pipeline exit code. Should check: if exit 0 (success), treat missing summary like COMPLETE and mark [~] → [x].

**Bug 2 — clear_milestone_state() unwired:**
- `tekhton.sh` lines 1130-1165: Post-commit milestone archival calls `archive_completed_milestone()` twice but never calls `clear_milestone_state()`. State file persists across runs and contaminates subsequent non-milestone tasks. Should add call after successful archival.
