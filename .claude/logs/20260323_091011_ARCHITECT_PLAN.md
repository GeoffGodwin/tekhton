### S-1: Extract `_indexer_resolve_cache_dir()` helper into `lib/indexer_helpers.sh`

**Observation source**: Drift log 2026-03-23

**Verified occurrences** (4 total — threshold crossed):

| File | Function | Lines | Has mkdir -p |
|------|----------|-------|--------------|
| `lib/indexer.sh` | `run_repo_map()` | 131–135 | yes |
| `lib/indexer.sh` | `invalidate_repo_map_cache()` | 272–275 | no |
| `lib/indexer_history.sh` | `_ensure_task_history_file()` | 24–28 | yes |
| `lib/indexer_history.sh` | `warm_index_cache()` | 59–63 | yes |

**Remediation**:

1. Add `_indexer_resolve_cache_dir()` to `lib/indexer_helpers.sh`. The function
   takes no arguments, reads `REPO_MAP_CACHE_DIR` and `PROJECT_DIR` from the
   environment, and prints the resolved absolute path on stdout:

   ```bash
   _indexer_resolve_cache_dir() {
       local cache_dir="${REPO_MAP_CACHE_DIR:-.claude/index}"
       if [[ "$cache_dir" != /* ]]; then
           cache_dir="${PROJECT_DIR}/${cache_dir}"
       fi
       echo "$cache_dir"
   }
   ```

2. Replace all 4 inline idioms with a call to the helper. Each call site that
   previously called `mkdir -p` continues to do so after the helper call —
   `_indexer_resolve_cache_dir()` resolves the path only; it does not create
   the directory.

   Replacement pattern (sites that had mkdir):
   ```bash
   local cache_dir
   cache_dir=$(_indexer_resolve_cache_dir)
   mkdir -p "$cache_dir" 2>/dev/null || true
   ```

   Replacement pattern (`invalidate_repo_map_cache` — no mkdir):
   ```bash
   local cache_dir
   cache_dir=$(_indexer_resolve_cache_dir)
   ```

3. `lib/indexer_history.sh` already declares `indexer_helpers.sh` as a
   dependency in its header comment. `lib/indexer.sh` sources
   `lib/indexer_helpers.sh`. No new sourcing changes are required.

**Scope**: 2 files, 4 call sites, 1 new function (~6 lines). No behavior change.
