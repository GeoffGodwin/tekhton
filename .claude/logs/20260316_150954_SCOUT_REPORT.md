## Suspected Root Cause Areas

1. **Pipeline loop insertion point** — After `run_stage_tester()` completes (tekhton.sh line 614), before `run_final_checks()` (line 638). Milestone checking and looping must happen here without disrupting the existing state save/archive flow.

2. **Milestone parsing** — Must extract numbered `#### Milestone N:` headings from CLAUDE.md with lenient formatting (users may vary headings, formatting, criteria placement). Current plan.sh shows basic milestone extraction at lines 325-328, but needs depth for acceptance criteria.

3. **Acceptance criteria automation** — Spec says "runs automatable criteria (`$TEST_CMD` passes, files exist, build gate passes)" but marking subjective criteria as `MANUAL`. Critical: don't try to LLM-evaluate human judgment. Tests must catch false-positive acceptance.

4. **State file format** — MILESTONE_STATE.md needs to track: current milestone number, status per milestone, transition history with timestamps. Must be both human-readable (for manual recovery) and machine-parseable by lib/milestones.sh.

5. **Auto-advance limit enforcement** — Cap at 3 milestones per run (tekhton.sh parameter). Must track milestone count and exit cleanly at boundary.

6. **Config injection point** — AUTO_ADVANCE_* flags need to appear in argument parsing (tekhton.sh lines 317-400) and config defaults (lib/config.sh lines 53-110).
