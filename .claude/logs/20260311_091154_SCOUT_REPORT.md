## Suspected Root Cause Areas

- **Missing planning state persistence**: Currently `lib/plan.sh` has no state save/restore mechanism. If the interview is interrupted via Ctrl+C, users must start over. Need to mirror `lib/state.sh` pattern but for planning workflow.

- **Planning config not in pipeline.conf**: Planning defaults are hard-coded in lib/plan.sh (lines 17-20) and not overridable via pipeline.conf. Must add planning section to pipeline.conf.example and integrate into config.sh load flow.

- **No resume detection in --plan block**: Unlike execution pipeline (tekhton.sh lines 240-284), the --plan block (lines 170-184) has no resume logic. Resuming after an interrupted interview requires re-running from scratch or manual file recovery.

- **Interview state not tracked**: run_plan_interview() doesn't save section index or progress. If interrupted mid-section, there's no way to resume at the same spot without re-interviewing earlier sections.

- **--init doesn't auto-set DESIGN_FILE**: The --init command (lines 100-166) creates a stub CLAUDE.md but doesn't check for existing DESIGN.md and auto-populate DESIGN_FILE in pipeline.conf. Manual editing required.
