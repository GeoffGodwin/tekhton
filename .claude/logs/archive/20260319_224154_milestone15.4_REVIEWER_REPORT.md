## Non-Blocking Notes
- `lib/finalize.sh` is 336 lines — 36 lines over the 300-line guideline. The ACP justifies this as a new extraction rather than overloading `hooks.sh` (already 305 lines), consistent with the codebase split pattern. A follow-on could move `_print_action_items` (~57 lines) to a `lib/finalize_helpers.sh` to bring both files under 300.
- ARCHITECTURE.md was not updated to include `lib/finalize.sh` in the Layer 3 library table. The ACP explicitly notes this is needed but it wasn't done. Update should describe: sourcing order, dependencies (hooks.sh, notes.sh, drift_cleanup.sh, milestone_ops.sh, milestone_archival.sh), and the V3 extension contract.
- `print_run_summary` is no longer called for null runs. The old early-exit path called `print_run_summary` before `exit 0`. In the new implementation, `print_run_summary` only runs inside `_hook_commit` after a successful commit — null runs (FINAL_CHECK_RESULT=1) skip the commit hook and thus skip the summary. Minor UX regression but not incorrect.
- `finalize_run` is always called with exit code 0 from tekhton.sh. The `finalize_run 1` failure path (hooks a-c + f only) is correct but currently dead code for M15. This is intentional — M16's outer loop will exercise it. Add a code comment noting this explicitly so the path isn't removed as unused code.

## Coverage Gaps
- No tests for `finalize_run()` hook sequence, `register_finalize_hook()`, or individual hook functions. The hook registry pattern (ordering, per-hook guards, warning-but-continuing on failure) is not test-covered. Recommend a test file `tests/test_finalize_run.sh` for at least: hook registration order, `finalize_run 0` runs all hooks, `finalize_run 1` skips success-only hooks, a failing hook doesn't abort the sequence.

- ACP: `lib/finalize.sh` (new file instead of adding to `lib/hooks.sh`) — ACCEPT — Sound architectural decision. hooks.sh is already 305 lines; adding ~300 more lines would make it ~600 lines. The extraction follows the established codebase pattern (drift.sh → drift_cleanup.sh + drift_artifacts.sh; agent.sh → agent_monitor.sh + agent_helpers.sh + agent_retry.sh). The API boundary is clean: hooks.sh retains utility functions, finalize.sh owns the orchestration layer. ARCHITECTURE.md update still needed (see Non-Blocking Notes).

## Drift Observations
- `lib/hooks.sh` (line ~1) and `lib/notes.sh` (line ~1) both exceed 300 lines (305 and 341 respectively). This predates M15.3 and is a systemic drift from the 300-line file ceiling — neither file was touched in this milestone but both are accumulating size debt that will eventually force extraction. Track for architect audit.
- `COMMIT_MSG` is assigned inside `_hook_commit` without a `local` declaration (intentionally global, used post-hook), but there is no comment explaining it is intentionally global. A reader may attempt to add `local` in a future refactor, breaking the banner output. A `# global: used by subsequent banner/logging` comment would prevent this.
