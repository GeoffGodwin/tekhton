## What Was Fixed

- **Dead code removal in `lib/agent_monitor.sh`**: Removed four unused module-level functions (`_ring_buffer_reset`, `_ring_buffer_add`, `_ring_buffer_dump`, `_check_line_for_api_errors`) that were defined but never called. The FIFO reader subshell manages its own inline ring buffer (`_rb`, `_rb_idx`, `_rb_size`) since variable assignments in subshells do not propagate to the parent process. Removed the orphaned call to `_ring_buffer_reset` at line 154. This saves ~80 lines and eliminates misleading code that suggested an alternate ring buffer implementation path that was never used. Added a comment explaining why inline variables are necessary in the FIFO subshell.

- **ERE alternation bug in `lib/errors.sh:260`**: Fixed pattern `'anthropic\|claude\|api\.anthropic'` → `'anthropic|claude|api\.anthropic'`. In ERE (extended regex with `grep -E`), the pipe `|` is already special and does not need escaping. The escaped form `\|` matches a literal pipe character, breaking the fallback API error detection branch in `classify_error`. This was a silent logic bug that broke detection for API errors not matching earlier patterns.

- **macOS sed portability in `lib/errors.sh:431`**: Fixed bearer token redaction from `'s/bearer [A-Za-z0-9_.-]*/bearer [REDACTED]/gI'` → `'s/[Bb][Ee][Aa][Rr][Ee][Rr] [A-Za-z0-9_.-]*/bearer [REDACTED]/g'`. The `gI` (global + case-insensitive) flag is GNU sed-only. POSIX/macOS BSD sed silently ignores unknown flags, meaning the bearer token redaction had no effect on macOS. Expanded the pattern to use character classes for case-insensitivity to maintain portability per CLAUDE.md.
