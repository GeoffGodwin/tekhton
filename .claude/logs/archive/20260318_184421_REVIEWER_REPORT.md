## Simple Blockers (jr coder)
- `lib/agent_monitor.sh`: The blocking-read code path (lines 210–221, active when `AGENT_ACTIVITY_TIMEOUT=0`) skips both the ring buffer update and real-time API error detection. When a user sets `AGENT_ACTIVITY_TIMEOUT=0` to disable the timeout, neither `_rb` is populated nor `_stream_api_error` is set, so `agent_last_output.txt` is empty and `_API_ERROR_DETECTED` stays false after agent exit. Error classification then has no stream context and no API hint, defeating both 12.2 features on that code path. Fix: duplicate the ring buffer append and the `case "$line"` API-error detection block into the `else` branch (blocking-read path) so both paths maintain identical 12.2 instrumentation.

## Non-Blocking Notes
- `lib/common.sh` `report_error()`: Only the `_transient_label` line uses `printf` with a closing `$_box_v` border character. All other content lines (`ERROR:`, empty separator, `message`, `Recovery:`) use bare `echo "${_box_v}  ..."` with no right border. The rendered block has an open right side on most lines, which looks unfinished in UTF-8 mode. Low priority cosmetic fix.
- `lib/agent_monitor.sh` / `lib/agent.sh`: The milestone spec says "redirect agent stderr to `$SESSION_TMP/agent_stderr.txt`." The actual implementation uses `> "$_fifo" 2>&1`, merging stderr into the FIFO stream. A dedicated stderr file is never written; instead a synthetic one-line hint is created only when `_API_ERROR_DETECTED=true`. For non-API errors that produce real stderr output, `agent_stderr.txt` remains empty and classification relies entirely on `agent_last_output.txt`. This works in practice (the ring buffer captures everything via `2>&1`) but diverges from the spec intent. Worth noting for Milestone 12.3 if per-stream log files are expected.

## Drift Observations
- `lib/agent_monitor.sh` lines 132–222: The timed-read and blocking-read branches inside the FIFO reader subshell have diverged — the timed branch carries ring buffer, API detection, and future feature hooks while the blocking branch is a stripped-down copy. Future features added to the timed branch will need to be mirrored manually. A refactor to extract a `_process_fifo_line()` helper called from both branches would eliminate this maintenance surface.
