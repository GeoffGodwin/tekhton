## Root Cause (bugs only)
The `test_context_accounting.sh` test creates a minimal `pipeline.conf` with only 3 keys (`PROJECT_NAME`, `CLAUDE_STANDARD_MODEL`, `ANALYZE_CMD`) and calls `load_config()`. However, `load_config()` validated 20 required keys and called `exit 1` when 17 were missing — killing the test subshell before context budget defaults could be checked.

The context budget defaults were placed AFTER the required-key validation, which meant they were never reached when the minimal config was used. Additionally, the required-key validation was overly strict — many keys have sensible defaults derivable from `CLAUDE_STANDARD_MODEL`.

## Files Modified
- `lib/config.sh` — Added defaults for execution-pipeline keys (model configs, turn limits, file paths) before validation. Reduced required-key set to `PROJECT_NAME`, `CLAUDE_STANDARD_MODEL`, `ANALYZE_CMD`. Moved context budget defaults before validation.
