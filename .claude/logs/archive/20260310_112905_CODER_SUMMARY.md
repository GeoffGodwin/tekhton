## Architecture Decisions

- **Conversational mode via `--append-system-prompt`**: The interview uses `claude --append-system-prompt` (not `-p`) to run in interactive/conversational mode. This preserves Claude's default system prompt while adding the interview instructions. The user types answers directly in the terminal.
- **INT trap for Ctrl+C resilience**: Rather than complex signal forwarding, we `trap 'true' INT` before the claude process and restore after. This lets claude handle SIGINT naturally while preventing bash from exiting before printing the DESIGN.md preservation message.
- **Config defaults in lib/plan.sh**: `PLAN_INTERVIEW_MODEL` and `PLAN_INTERVIEW_MAX_TURNS` are set with env-var overridable defaults. This seeds forward for Milestone 6's config integration without requiring pipeline.conf to exist (--plan runs before config is loaded).
- **Log file captures metadata, not transcript**: The session log captures the system prompt, model, timestamps, and DESIGN.md status. The full interactive transcript is visible in the user's terminal and maintained by claude CLI's session history. This avoids the complexity of `script`/`tee` which can interfere with terminal interactivity.
