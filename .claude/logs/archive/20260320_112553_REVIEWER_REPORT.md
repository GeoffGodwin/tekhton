## Non-Blocking Notes
- `report_orchestration_status()` in `common.sh` reads `MAX_AUTONOMOUS_AGENT_CALLS` as a global instead of receiving it as a fifth parameter. The spec signature is `report_orchestration_status(attempt, max, elapsed, agent_calls)` — `max` refers to pipeline attempt max. Agent call ceiling leaks in as a global side-channel. Consider adding a fifth parameter or making the dependency explicit via a comment.
- `finalize_summary.sh` JSON string escaping via `sed 's/"/\\"/g'` only escapes double-quotes, not backslashes. A milestone ID or task string containing a backslash would produce invalid JSON. Use `sed 's/\\/\\\\/g; s/"/\\"/g'` for robustness, matching the pattern already used for `files_json` path escaping in the same file.
- `lib/metrics.sh` remains ~395 lines — a pre-existing violation that predates M16. Extraction of `_avg_field`, `_scout_accuracy`, and `summarize_metrics` into `metrics_dashboard.sh` would bring it under the ceiling. Not blocking this merge.
- `_run_auto_advance_chain()` in `orchestrate_helpers.sh` calls `run_complete_loop` recursively (line 44). Recursion depth is bounded only by `AUTO_ADVANCE_LIMIT` and the function stack. This is the pre-existing pattern from M16's original design. No action required, but a comment noting the recursion depth bound would prevent future confusion.

## Coverage Gaps
