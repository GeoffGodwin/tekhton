## Non-Blocking Notes
- `lib/health.sh` (418 lines) and `lib/health_checks.sh` (581 lines) both exceed the 300-line soft ceiling. Both files are logically cohesive — health_checks.sh is inherently long with 5 independent dimension functions. Consider splitting health_checks.sh into `health_checks_code.sh` and `health_checks_infra.sh` in a future cleanup pass.
- `assess_project_health()` and `reassess_project_health()` duplicate the full dimension-check loop (lines 70-95 vs 172-196 in health.sh). The behavioral difference (read prev baseline before overwriting it) is intentional and documented, but bug fixes must be applied in two places. A shared `_run_dimension_checks()` helper returning all five scores would eliminate the duplication without changing behavior.
- `emit_dashboard_health()` is called inside `_hook_causal_log_finalize` (hook 4), which runs before `_hook_health_reassess` (hook 11). When `HEALTH_REASSESS_ON_COMPLETE=true`, the dashboard writes stale pre-reassessment data for the current run. Consider adding a second `emit_dashboard_health` call at the end of `_hook_health_reassess`, or reorder so the health hook fires before `_hook_causal_log_finalize`'s dashboard emission.
- `local ratio=$(( ... ))` and similar `local var=$((expr))` patterns in health_checks.sh and config.sh are consistent with the existing codebase-wide pattern (same pattern in common.sh, context.sh, agent_helpers.sh, etc.). No shellcheck action needed.
