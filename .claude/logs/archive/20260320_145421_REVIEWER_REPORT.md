## Non-Blocking Notes
- `_find_source_files` uses `git ls-files` in git repos (returns all tracked files at all depths) while the non-git fallback uses `find -maxdepth 2`. The spec and the `_count_source_files` docstring both say "top 2 levels" — the git path over-counts source files in deep repos, inflating confidence. Impact is minor (over-counting → higher confidence, which is usually correct direction) but worth aligning.
- `_DETECT_EXCLUDE_DIRS` contains literal dots (`.git`, `.dart_tool`, `.next`, etc.) used directly in `grep -Ev` ERE patterns without escaping. A dot in ERE matches any character, so `.git` would also match `XgitY`-style directory names. In practice harmless, but a pedantic shellcheck / reviewer would flag it.
- `detect_frameworks` labels Node.js frameworks (react, next.js, vue, angular, svelte, express, fastify) with `typescript` or `javascript` lang based on framework convention, not on what the detection engine actually found. A TypeScript+Express project would be labeled `javascript|express` while being detected as `typescript` in languages. Consider passing detected language through or using `node` as a neutral label. Minor cosmetic inconsistency.
- `_extract_json_keys` is defined in `detect.sh` but called by `detect_commands.sh`. The dependency is documented in the header comment, and the test correctly sources `detect.sh` first (with an explanatory comment). No action needed — just noting the cross-file function dependency for future readers.
