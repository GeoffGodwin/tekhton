### 1. Harden `prompts/plan_generate.prompt.md` to use explicit `{{PROJECT_DIR}}` path

**File**: `prompts/plan_generate.prompt.md`

**Current state**: Line 17 says "Generate a complete `CLAUDE.md` file and write it to disk in the **current working directory**." Line 74 says "Use the Write tool to create `CLAUDE.md` in the current working directory."

**Problem**: The agent is told to write relative to CWD. This is correct today — `tekhton.sh` captures `PROJECT_DIR=$(pwd)` at startup and the `--plan` block never changes directory — but it is an implicit assumption. If any future code in the `--plan` flow introduces a `cd`, the agent would silently write CLAUDE.md to the wrong location, with no error and no warning.

**Fix**: Replace both occurrences of "current working directory" in `prompts/plan_generate.prompt.md` with the explicit path `{{PROJECT_DIR}}/CLAUDE.md`. The `render_prompt()` engine already supports this: it expands all exported shell variables via `${!var_name:-}` (lib/prompts.sh line 78), and `PROJECT_DIR` is exported at tekhton.sh line 85 before the `--plan` block runs. No other files need to change.

- Line 17: "write it to disk in the current working directory" → "write it to disk at `{{PROJECT_DIR}}/CLAUDE.md`"
- Line 74: "Use the Write tool to create `CLAUDE.md` in the current working directory" → "Use the Write tool to create the file at the path `{{PROJECT_DIR}}/CLAUDE.md`"

**Bounded**: Two line edits in one file. No logic changes.

### 2. Add `PROJECT_DIR` to the template variable table in `CLAUDE.md`

**File**: `CLAUDE.md` (Tekhton repo), section "Template Variables (Prompt Engine)"

**Current state**: `PROJECT_DIR` is not listed in the template variable table, even though it is exported by `tekhton.sh` and available to all `render_prompt()` calls. After fix #1 above, it will be actively used in `prompts/plan_generate.prompt.md`.

**Fix**: Add a row to the template variables table:

| Variable | Source |
|----------|--------|
| `PROJECT_DIR` | `pwd` at tekhton.sh startup |

**Bounded**: One table row added to CLAUDE.md.
