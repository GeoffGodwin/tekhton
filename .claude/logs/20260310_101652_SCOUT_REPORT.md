## Relevant Files

- `lib/plan.sh` — Orchestration library created in Milestone 1; contains `run_plan()` function and project type selection. Interview will be wired into `run_plan()` after type selection (after line 90).
- `lib/agent.sh` — Agent invocation wrapper; currently uses `-p` (batch mode) for all agents. Interview will need conversational mode invocation (without `-p` or with different flags).
- `lib/common.sh` — Logging utilities (`log()`, `success()`, `warn()`, `header()`). Used by interview for status messages.
- `lib/prompts.sh` — Prompt template renderer. Currently loads `{{VAR}}` substitution; interview prompts will follow same pattern.
- `tekhton.sh` — Entry point; sources `lib/plan.sh` for `--plan` flag (lines 170-174). Early-exit before config load. Interview stage will be sourced and called from within `run_plan()`.
- `templates/plans/web-app.md` (and 6 other project type templates) — Design doc templates with section headings and guidance comments. Interview agent will read these and ask questions section-by-section.
- `stages/coder.sh` (and other existing stages) — Pattern reference for stage implementation (context setup, agent invocation, logging).

## Suspected Root Cause Areas

- **Interview invocation strategy** — `lib/agent.sh` currently uses `-p` (batch mode); conversational mode requires different claude CLI invocation (possibly `claude` without `-p`, or with stdin passthrough). Will need to determine correct flags for interactive conversation + logging.
- **DESIGN.md progressive writing** — Interview agent must write DESIGN.md incrementally during conversation (not accumulate and write at the end). Prompt template must instruct agent to use `write()` or similar between questions.
- **Conversational loop control** — Unlike batch agents that exit after completion, interview needs to loop: ask → read user input → write section → move to next. This is a control-flow departure from `run_agent()` pattern.
- **Section tracking** — Interview needs to track which sections are done vs. pending. This state may need to persist if interrupted (for Milestone 6 resume support, but Milestone 2 only requires Ctrl+C preservation of partial DESIGN.md).
- **Log file handling** — Interview produces conversational logs (question-answer pairs) instead of structured JSON. Logging must capture the full conversation in a readable format.
