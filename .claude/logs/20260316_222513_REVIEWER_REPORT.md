## Non-Blocking Notes
- `stages/cleanup.sh:27` — `[ "$unresolved" -le "$threshold" ] 2>/dev/null` redirects stderr from the `[` arithmetic comparison, which never produces stderr output. The `2>/dev/null` is harmless but misleading; remove it for clarity.
- `lib/notes.sh:264,287` — `sed -i "0,/pattern/s//replacement/"` uses GNU sed's `0,` address (first-match-only range). This is not portable to macOS's BSD sed. Since the project targets Linux/WSL primarily this is acceptable, but worth a comment documenting the GNU sed dependency alongside the existing shellcheck disable comments.
- `lib/notes.sh:270` — `grep -q "$basename_mod"` in `_resolve_cleanup_by_file_changes` passes an unquoted variable that could contain regex metacharacters (e.g., a filename like `foo.sh` where `.` matches any char). Use `grep -qF "$basename_mod"` for literal matching, consistent with the `grep -qxF` usage at `stages/cleanup.sh:126`.
