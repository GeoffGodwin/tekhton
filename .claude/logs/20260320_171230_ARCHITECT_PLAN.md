**Item B-1: Latent arithmetic-on-empty-string bug in `metrics_dashboard.sh`**

This item does not fit cleanly into Staleness, Dead Code, or Naming — it is a pre-existing latent bug. Route to jr coder alongside S-1 since both affect `metrics_dashboard.sh`.

- **File:** `lib/metrics_dashboard.sh`, lines 163–164 and 187–191
- **Finding:** `val`, `est`, and `actual` are assigned via `grep ... || true`. When grep produces no match, the variable is assigned an empty string. Subsequent `$(( sum + val ))` and `$(( est - actual ))` calls then fail with a bash arithmetic syntax error (`integer expression expected`). The `|| true` prevents the grep from killing the pipeline but does not coerce the result to `0`.
- **Action:** Apply parameter default expansion at each arithmetic site: change `$(( sum + val ))` to `$(( sum + ${val:-0} ))`, and similarly `$(( est - actual ))` to `$(( ${est:-0} - ${actual:-0} ))`. Do not change any other logic. This is a minimal, self-contained fix.
- **Route:** Jr coder
