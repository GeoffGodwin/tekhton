## What Was Implemented
- Fixed TMPDIR variable shadowing system `$TMPDIR` environment variable in test file — renamed to `TEST_TMP`

- Fixed confusing timeout warning in `lib/agent.sh` — when `AGENT_TIMEOUT=0` (outer timeout disabled) and the activity timeout triggers, the null-run message now correctly reports "activity-timed out after Ns of silence" instead of the misleading "timed out after 0s"

## Root Cause (bugs only)
1. **TMPDIR shadowing**: Test used `TMPDIR` as a local temp directory variable, which shadows the system `$TMPDIR` environment variable used by `mktemp` and other tools.
2. **PATH misleading syntax**: Bash `VAR=value func` where `func` is a shell function does NOT create a temporary environment (unlike external commands). PATH was being permanently modified and accumulating duplicate `MOCK_BIN` entries on each call.
3. **Timeout message confusion**: The null-run detection at line 361 always reported `${_timeout}s` (the outer AGENT_TIMEOUT value), but when AGENT_TIMEOUT=0, the actual killer was the activity timeout. The message now branches on `_was_activity_timeout` to report the correct timeout type and duration.
