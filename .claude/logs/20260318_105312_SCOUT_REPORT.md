## Suspected Root Cause Areas

- **Phase 1 missing functions**: `check_milestone_size()` and `split_milestone()` are called in stages/coder.sh (lines 79, 82) but not defined anywhere; pipeline fails if MILESTONE_MODE is active and scout estimate exceeds threshold
- **Phase 2 missing null-run handler**: tekhton.sh detects null-run but has no auto-split recovery path; currently exits with "null run" warning instead of attempting to split and retry
- **Phase 2 missing attempt tracking**: No functions to record or retrieve prior milestone split attempts; splitter prompt cannot reference failure history to avoid re-producing same oversized split
- **Sub-milestone numbering**: When split_milestone() replaces a milestone N with N.1, N.2, N.3, the archival code must correctly handle both original numbering (N) and new numbering (N.K); `_replace_milestone_block()` may need updates to handle sub-milestone insertion
- **Recursion depth limiting**: No enforcement of MILESTONE_MAX_SPLIT_DEPTH in the null-run auto-split retry path; if split fails repeatedly, could exceed configured depth without stopping
