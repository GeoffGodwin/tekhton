## Staleness Fixes

### SF-1: Move `# shellcheck disable=SC2034` inside `_rule_unknown()` function body

**File**: `lib/diagnose_rules.sh`, line 300
**Observation**: The disable comment sits above the function definition line rather than
inside the function body. Shellcheck correctly applies it to the entire function body
in this position, so behavior is correct and the file passes shellcheck clean. However,
future readers may interpret it as applying only to the `_rule_unknown()` definition
line itself (which cannot produce SC2034), creating confusion about why it's there.

**Action**: Move `# shellcheck disable=SC2034` to inside the function body, directly
above the `DIAG_CLASSIFICATION=` assignment on what is currently line 302. No change
to suppression scope — shellcheck treats both placements as covering the function body.
Remove the comment from line 300 and insert it at the new position.

```
_rule_unknown() {
    # shellcheck disable=SC2034  # DIAG_* are globals read by the caller
    DIAG_CLASSIFICATION="UNKNOWN"
    ...
```

## Naming Normalization

### NN-1: Normalize conditional brackets in `clear_pipeline_state()` to `[[ ]]`

**File**: `lib/state.sh`, lines 115 and 120
**Observation**: `clear_pipeline_state()` uses POSIX single-bracket `[ -f ]` in both
its original guard (line 115) and the M17-added guard (line 120). The project standard
is `[[ ]]` throughout all other `.sh` files.

**Action**: Replace both occurrences in `clear_pipeline_state()`:
- Line 115: `if [ -f "$PIPELINE_STATE_FILE" ]` → `if [[ -f "$PIPELINE_STATE_FILE" ]]`
- Line 120: `if [ -f "$failure_ctx" ]` → `if [[ -f "$failure_ctx" ]]`

Run `shellcheck lib/state.sh` and `bash -n lib/state.sh` to confirm no regressions.
