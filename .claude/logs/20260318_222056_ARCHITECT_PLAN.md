## Simplification

### S-1: Extract shared `_print_box_line()` from duplicated nested functions in `lib/common.sh`

**Location:** `lib/common.sh` lines 77–86 (`_box_line` inside `report_error`) and lines 132–141 (`_rbox_line` inside `report_retry`)

**Observation addressed:** DRIFT-2026-03-18 — `_box_line` and `_rbox_line` are nested functions with identical `printf`/fallback `echo` implementations. Only the name differs. Silent divergence risk on any future edit.

**Root cause:** Milestone 13.1 added `report_retry()` by copy-pasting the box-drawing inner function from `report_error()` and renaming it `_rbox_line` to avoid an apparent name collision. Because bash nested functions are not lexically scoped — they enter global scope on first call — the rename was unnecessary and created a second definition of the same logic.

**Remediation:** Extract a single module-level helper `_print_box_line()` with the shared signature:
```
_print_box_line(box_v, box_w, content)
```
Remove the `_box_line` definition from inside `report_error()`.
Remove the `_rbox_line` definition from inside `report_retry()`.
Replace all four call sites (two per function: the content-line path and the empty-line path) with calls to `_print_box_line`.

The helper must be defined before `report_error()` and `report_retry()` in the file. It should not use globals for `_box_v` and `_box_w` — take them as positional parameters so the function is self-contained and testable.

**Scope:** `lib/common.sh` only. No other files change. No behavior change.
