- Fixed `append_drift_observations()` in `lib/drift.sh` to join multi-line observations into single entries (same pattern as `append_nonblocking_notes()`)
- Consolidated 5 corrupted drift log entries into 1 resolved entry in `DRIFT_LOG.md`

## Root Cause (bugs only)
1. **TESTER_REPORT.md**: Git-tracked version referenced functions `_extract_project_name()` and `_extract_milestones()` that were removed from `lib/plan.sh` (inlined into `_display_milestone_summary()`). The file was also deleted from the working tree.
2. **NON_BLOCKING_LOG.md**: Items marked `[x]` by the pipeline's `_resolve_addressed_nonblocking_notes()` were left in the `## Open` section instead of being moved to `## Resolved`.
3. **gates.sh**: `run_completion_gate()` was changed to only handle the two-line status format (`## Status\nVALUE`) but the standard coder template uses single-line format (`## Status: VALUE`), causing the completion gate to always fail with "no clear Status field".
4. **drift.sh**: `append_drift_observations()` split each line of a multi-line observation into separate entries (each with its own date/task prefix), corrupting the drift log. Root cause: unlike `append_nonblocking_notes()`, it lacked continuation-line joining logic.
