### Depth Scoring Algorithm

Pseudo-code for `_score_section_depth()`:

```
function _score_section_depth(content):
    score = 0
    if content has "###" sub-headings:
        score += 2
    if content has "|...|...|" tables:
        score += 1
    if content has "```" code blocks:
        score += 1
    if line_count(content) >= 5:
        score += 1
    return score  # 0-5
```

A score < 2 (shallow) means: missing structure (sub-sections) AND short content OR lacks examples.

### Exit Strategy

The follow-up interview is part of the completeness loop's multi-pass approach:
- Pass 1: Initial completeness check
- Pass 2+: Follow-up interviews (max 3 passes total)
- After max passes, user can skip and continue with incomplete DESIGN.md
- Each pass: if completeness check passes, exit loop; else offer follow-up

Milestone 4 adds the **depth dimension** to completeness checks: sections must pass BOTH "is not empty" AND "is not shallow" criteria.
