# Scout Report — Context Accounting Test Failure

## Test Failure Summary
The test `test_context_accounting.sh` fails at the "Context config defaults" section. The test creates a minimal `pipeline.conf` with only `PROJECT_NAME`, `CLAUDE_STANDARD_MODEL`, and `ANALYZE_CMD`, then calls `load_config()` expecting it to set context defaults (`CONTEXT_BUDGET_PCT=50`, `CHARS_PER_TOKEN=4`, `CONTEXT_BUDGET_ENABLED=true`).

However, `load_config()` exits early due to validation of required keys (lines 105-126 in `lib/config.sh`) before reaching the context defaults section (lines 159-162).

- `tests/test_context_accounting.sh:389-462` — Test section that creates minimal config and calls load_config()

## Suspected Root Cause Areas
- `lib/config.sh` lines 105-126: Validation of required keys happens BEFORE optional defaults are set. The validation includes keys like `REQUIRED_TOOLS`, `CLAUDE_CODER_MODEL`, `CODER_MAX_TURNS`, `TEST_CMD`, `PIPELINE_STATE_FILE`, `LOG_DIR`, `CODER_ROLE_FILE`, `REVIEWER_ROLE_FILE`, `TESTER_ROLE_FILE`, `JR_CODER_ROLE_FILE`, `PROJECT_RULES_FILE` that are not provided by the test's minimal config.
- Solution: Set defaults for required keys BEFORE validation check, so variables are already defined when validation runs.
