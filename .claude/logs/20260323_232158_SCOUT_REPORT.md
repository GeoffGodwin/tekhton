## Suspected Root Cause Areas

- **tekhton.sh:1003** — The condition `[[ ! -d "$local_dash_dir" ]]` means `init_dashboard()` only runs on first startup. Once `.claude/dashboard/` exists (after first run creates `data/` subdirectory), the function is never called again. The static UI files (added in M14) are only copied during `init_dashboard()`, so they never appear in existing projects.
- **lib/dashboard.sh:65-79** — The `_copy_static_files()` function is private (underscore prefix) and only called from within `init_dashboard()`. It's never called on subsequent runs, so template updates don't propagate to deployed dashboards.
- **Deployment logic** — The fix requires ensuring static files are synced on every startup, not just first creation. This can be done by either (a) refactoring `_copy_static_files()` into a public callable function, or (b) creating a new public wrapper like `ensure_static_ui_files()` that's called separately from `init_dashboard()`.
