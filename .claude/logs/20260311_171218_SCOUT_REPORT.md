# Scout Report — Fix 4 Bugs from Milestone 2

## Relevant Files
- `tests/test_agent_fifo_invocation.sh` — Contains 3 bugs: single-bracket conditional (line 260), misleading PATH temporary assignments (lines 125, 155, 185, 217, 238, 241), TMPDIR shadowing system env var (line 17)
- `lib/agent.sh` — Contains 1 bug: timeout warning message outputs confusing `${_timeout}s` value when AGENT_TIMEOUT=0 (line 361)

## Key Symbols
- `TMPDIR` — test_agent_fifo_invocation.sh line 17 (should be renamed to TEST_TMP)
- `PATH="${MOCK_BIN}:${PATH}" run_agent` — test_agent_fifo_invocation.sh lines 125, 155, 185, 217, 238, 241 (misleading syntax)
- `if [ "$FAIL" -ne 0 ]` — test_agent_fifo_invocation.sh line 260 (should be `[[ ]]`)
- `warn "[$label] NULL RUN DETECTED — agent timed out after ${_timeout}s."` — lib/agent.sh line 361 (message unclear when timeout=0)

## Suspected Root Cause Areas
- **test_agent_fifo_invocation.sh line 17**: TMPDIR variable name shadows `$TMPDIR` system environment variable. Rename to `TEST_TMP` or `_TMPDIR` to avoid shadowing.
- **test_agent_fifo_invocation.sh lines 125, 155, 185, 217, 238, 241**: Use of `PATH="${MOCK_BIN}:${PATH}" run_agent` appears to create a temporary environment but actually modifies PATH permanently in the current shell (bash shell-function semantics, not external-command semantics). Should export PATH once at the top of the test and prepend MOCK_BIN globally instead.
- **test_agent_fifo_invocation.sh line 260**: Single-bracket conditional `if [ "$FAIL" -ne 0 ]` violates project bash style requirement (double-bracket conditionals). Should be `if [[ "$FAIL" -ne 0 ]]`.
- **lib/agent.sh line 361**: Timeout warning message outputs `${_timeout}s` which is the literal string `0` (confusing) when `AGENT_TIMEOUT=0` (outer timeout disabled). Message should be clearer about whether it's an activity timeout vs a global timeout.
