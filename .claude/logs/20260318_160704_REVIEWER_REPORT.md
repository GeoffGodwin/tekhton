## Non-Blocking Notes
- `lib/agent_monitor.sh:237-244` — The ring-buffer dump block (inside the FIFO reader subshell) uses `local` declarations inside a `{...}` compound group that is NOT inside a function. In bash 4, `local` outside a function always fails with "local: can only be used in a function". With `set -e` active in the subshell, this causes the entire `{...}` block to abort at the first `local` line; the `|| true` at the end of the block (line 249) suppresses the error but the ring buffer is never actually written to `agent_last_output.txt`. The comment added this run correctly explains the rationale for avoiding a function call, but the `local` usage is the underlying issue. The fix is to remove the `local` declarations and use plain assignments (`_rb_total=...`), since the variables are already subshell-local by virtue of being in the `(...)` subshell. Pre-existing issue not introduced by this coder — flagging here since the new comment draws attention to this block.
