## Suspected Root Cause Areas

- **Unbounded outer loop failure**: Current --complete mode has MAX_PIPELINE_ATTEMPTS counter that treats both success and failure the same way. The counter should only increment on consecutive failures without forward progress.
- **No quota awareness**: Agent retries fail hard on rate limits instead of pausing gracefully. The claude CLI returns 429/rate-limit errors but the pipeline has no detection or recovery mechanism.
- **Inaccurate progress detection**: Current fallback to git diff alone misses work like security scans that don't change files. Causal log (from M13) provides richer signals but isn't integrated into orchestrate_helpers.
- **Stale tests after archival**: test_milestone_15_2_2_2_migration.sh checks CLAUDE.md content that gets moved to MILESTONE_ARCHIVE.md. Tests designed against one milestone's structure fail silently when that milestone is archived in a later run.
- **No quota statistics**: Pipeline doesn't track pause duration, pause count, or quota status — Watchtower Live Run has no visibility into quota pauses.
