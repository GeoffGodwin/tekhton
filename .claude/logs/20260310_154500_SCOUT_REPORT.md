# Scout Report: Milestone 4 — CLAUDE.md Generation Agent

## Relevant Files

- **`lib/plan.sh`** — Orchestration library. Contains `run_plan()` function (line 266) which is the planning phase entry point. Currently has 3 steps: project type selection, interview, and completeness check. Generation needs to be wired after line 280 as step 4.

- **`stages/plan_interview.sh`** — Interactive interview stage implementation. Shows the pattern for conversational mode agent invocation with `--append-system-prompt`. Generation will be batch mode (`-p` flag) instead. Already sources `render_prompt()` for template substitution.

- **`prompts/plan_interview.prompt.md`** — Interview agent system prompt template. Shows use of `{{VAR}}` substitution and conditional blocks `{{IF:VAR}}...{{ENDIF:VAR}}`. Generation prompt should follow the same pattern but focus on CLAUDE.md structure.

- **`prompts/plan_interview_followup.prompt.md`** — Follow-up prompt shows how to read existing file content via `{{DESIGN_CONTENT}}`. Generation prompt will similarly need to read `{{DESIGN_CONTENT}}` and synthesize it into CLAUDE.md sections.

- **`lib/agent.sh`** — Batch-mode agent wrapper. `run_agent(name, model, turns, prompt, logfile)` invokes claude with `-p "$prompt"`. Used by coder/reviewer/tester stages. Generation stage will use this directly.

- **`lib/prompts.sh`** — Template engine. `render_prompt(template_name)` reads from `prompts/<name>.prompt.md`, substitutes `{{VAR}}` with shell globals, strips `{{IF:VAR}}...{{ENDIF:VAR}}` blocks when VAR is empty.

- **`lib/common.sh`** — Shared logging functions (`header()`, `log()`, `warn()`, `success()`, `error()`). Generation stage will use these for output messaging.

- **`tekhton.sh`** (lines 170-176) — Entry point. The `--plan` early-exit block sources `plan_interview.sh` at line 174. Needs to also source new `plan_generate.sh` file.

- **`stages/coder.sh`** (lines 54-59, 206-214) — Reference implementation of how to invoke agents via `run_agent()`. Shows context block building (ARCHITECTURE_BLOCK, GLOSSARY_BLOCK, etc.) and prompt rendering. Generation can follow the same pattern but simpler (only DESIGN.md as context).

## Key Symbols

- `run_plan()` (lib/plan.sh:266) — Planning phase orchestration; calls interview, completeness check; must call generation next
- `render_prompt(template_name)` (lib/prompts.sh:29) — Template substitution engine; generation will call this
- `run_agent(label, model, turns, prompt, logfile)` (lib/agent.sh:46) — Batch-mode agent invoker; generation uses this for CLAUDE.md synthesis
- `run_plan_interview()` (stages/plan_interview.sh:26) — Conversational interview reference; generation differs by using batch mode
- `run_plan_completeness_loop()` (lib/plan.sh:205) — Completeness check; generation runs after this returns 0

## Suspected Root Cause Areas

- **Lack of generation stage orchestration** — `run_plan()` has a comment at line 282-284 saying generation and file output are future milestones; they must be wired into the function after completeness check succeeds.

- **Missing generation prompt template** — No `prompts/plan_generate.prompt.md` exists; must be created with system instructions for synthesizing CLAUDE.md from DESIGN.md (rules, milestones with acceptance criteria, architecture guidelines, testing strategy).

- **Missing generation stage implementation** — No `stages/plan_generate.sh` exists; must implement `run_plan_generate()` function that reads DESIGN.md as context, renders the generation prompt, invokes `run_agent()` in batch mode, and produces CLAUDE.md.

- **Generation not sourced in tekhton.sh** — The `--plan` block sources `plan_interview.sh` but not a future `plan_generate.sh`; must add sourcing at line 174.

- **No generation config defaults** — Unlike interview phase (PLAN_INTERVIEW_MODEL, PLAN_INTERVIEW_MAX_TURNS), generation will need its own model and turn limit exports. Should be optional (defaulting to CLAUDE_PLAN_MODEL) but available for config override.
