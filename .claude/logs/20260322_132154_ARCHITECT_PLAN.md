## Simplification

### S1 — Fix `archive_all_completed_milestones` missing DAG path (`lib/milestone_archival.sh:138–171`)

**Observation:** `archive_all_completed_milestones` discovers done milestones by
grepping CLAUDE.md for `#### [DONE] Milestone` markers. In DAG mode these markers
do not exist — milestones live in individual files under `.claude/milestones/`.
The function silently returns 0 (zero archived), while the per-milestone
`archive_completed_milestone` correctly handles the DAG path.

**Caller:** `tekhton.sh:892` — invoked during finalization.

**Fix:** Add a DAG path at the top of `archive_all_completed_milestones`:
1. If `MILESTONE_DAG_ENABLED=true` and `has_milestone_manifest` returns 0,
   load the manifest (if not already loaded), iterate `_DAG_IDS[]`, and for each
   entry whose status is `done` call `archive_completed_milestone "$num" "$claude_md"`.
   Increment `archived_count` normally.
2. The existing CLAUDE.md grep path is unchanged (inline mode fallback).

No new functions needed — reuses the same `archive_completed_milestone` delegate
that already has the correct DAG path, and the same `dag_get_status` / `load_manifest`
calls already used in `milestones.sh`.

**Routing:** Senior coder (new conditional logic branch).

### S2 — Extract DAG helper wrappers from `lib/milestones.sh` to bring it under 300 lines

**Observation:** `lib/milestones.sh` is 410 lines, violating the 300-line Non-Negotiable
Rule. The three DAG-aware wrapper functions added in Milestone 1 are the primary
contributors: `get_milestone_count` (lines 177–198), `get_milestone_title`
(lines 200–223), `is_milestone_done` (lines 225–249), plus `parse_milestones_auto`
(lines 109–175).

**Fix:** Create `lib/milestone_dag_helpers.sh` and move these four functions there:
- `parse_milestones_auto`
- `get_milestone_count`
- `get_milestone_title`
- `is_milestone_done`

Update `milestones.sh` to source `milestone_dag_helpers.sh` at the top (same
pattern as `milestone_archival.sh` and `milestone_ops.sh`). Update `tekhton.sh`
to source the new file alongside the other milestone modules.

After extraction, `milestones.sh` drops to ~230 lines (within the 300-line limit).
All callers of the four moved functions are unchanged — they reference them by name
and the source chain makes them available identically.

**Routing:** Senior coder (code extraction with sourcing wiring changes).

### S3 — Split `stages/init_synthesize.sh` to bring it under 300 lines

Source the helpers file from within the stage file.
