## Suspected Root Cause Areas

1. **Shell safety directive missing**: Both `lib/milestone_split.sh` and `lib/milestone_ops.sh` lack the standard `set -euo pipefail` preamble. While they inherit this setting via `source` from `tekhton.sh`, this diverges from project convention (all existing `.sh` files have it explicitly) and creates fragile dependency — if sourced independently, silent failures could occur.

2. **Incomplete `set -euo pipefail` enforcement**: The CLAUDE.md security audit (Milestone 0) requires all `.sh` files to pass `bash -n` syntax check and `shellcheck` without warnings. New library files need verification.

3. **File size approaching limit**: `lib/milestones.sh` is 273 lines; the code quality guideline ceiling is 300. The newly created `lib/milestone_ops.sh` (284 lines) also approaches this limit. Future milestone work (12+) may require splitting these further.

4. **Three split orchestration paths**: Pre-flight sizing, null-run detection, and turn-limit minimal output each invoke `handle_null_run_split()` with recursive `run_stage_coder()` calls. Recursion depth is guarded by `MILESTONE_MAX_SPLIT_DEPTH=3`, but edge cases (split fails at max depth, cannot-split scenarios) need validation.

5. **State persistence across recursive calls**: When a milestone splits and coder re-runs recursively, `_CURRENT_MILESTONE` and `TASK` globals are updated in-place (not subshelled). Resume logic must correctly track which depth level was interrupted.
