## Suspected Root Cause Areas

1. **stages/plan_interview.sh line 91**: `wc -l < "${PROJECT_DIR}/DESIGN.md"` produces leading whitespace on macOS/BSD platforms. When used in string context (line 91), causes failure. **Fix: pipe through `| tr -d ' '`** to strip leading spaces.

2. **stages/plan_interview.sh line 102**: Same `wc -l` whitespace issue in variable assignment (`line_count=$(wc -l < ...)`). Leading spaces break numeric comparison with `[ "$exit_code" -ne 0 ]` on line 104. **Fix: add `| tr -d ' '`**.

3. **tests/test_plan_interview_stage.sh line 51**: Single-quoted `export PATH=\'${mock_bin}:${PATH}\'` in bash -c subshell prevents `${mock_bin}` variable expansion. Results in literal string instead of actual path, causing `mkdir`, `cat`, `date` to fail. **Fix: use double quotes with proper escaping: `export PATH=\"${mock_bin}:\${PATH}\"`** so variables expand correctly.
