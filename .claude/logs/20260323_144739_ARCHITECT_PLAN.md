### SF-1: `lib/detect_ci.sh` — `_classify_ci_command` output format undocumented and emitters are 6-field

**Location**: `lib/detect_ci.sh` lines 197–241

**Issue**: The `detect_ci_config` function header (line 16) correctly documents a
7-field output format: `CI_SYSTEM|BUILD_CMD|TEST_CMD|LINT_CMD|DEPLOY_TARGET|_LANG|CONFIDENCE`.
However, the three `echo` statements inside `_classify_ci_command` (lines 215, 227,
238) emit only 6 fields — they omit the `_LANG` placeholder present in the
`_detect_dockerfile_langs` emitter. The `_classify_ci_command` function comment
(lines 197–198) also says nothing about the output format, making the mismatch
invisible to future contributors.

**Fix**:
1. Add an `# Output:` line to the `_classify_ci_command` function comment documenting
   the full 7-field format.
2. Insert an empty `_LANG` placeholder into all three echo statements:
   - Test branch (line 215): `"${ci_system}||${cmd}||||high"` → add one more empty field
   - Lint branch (line 227): `"${ci_system}|||${cmd}|||high"` → add one more empty field
   - Build branch (line 238): `"${ci_system}|${cmd}|||||high"` → add one more empty field
3. Verify `detect_ci_config` header at line 16 matches — it already shows 7 fields
   and requires no change.

## Naming Normalization

Route to: **jr coder**

### NN-1: `lib/detect_ai_artifacts.sh:81` — `dir_name` used for a file path

**Location**: `lib/detect_ai_artifacts.sh` line 81 (inside the `_KNOWN_AI_FILES` loop)

**Issue**: The outer `local` declaration at line 64 introduces `dir_name` in the
context of the `_KNOWN_AI_DIRS` loop where it correctly refers to a directory name.
The `_KNOWN_AI_FILES` loop at line 80–85 reuses the same `dir_name` variable via
`IFS='|' read -r dir_name tool_name <<< "$entry"`, but each entry in `_KNOWN_AI_FILES`
holds a file path (e.g., `CLAUDE.md`, `.cursorrules`), not a directory. The variable
name misleads: a reader must follow the `_KNOWN_AI_FILES` array definition to know
that `dir_name` holds a file path here.

**Fix**: Change the `read` at line 81 from:
```
IFS='|' read -r dir_name tool_name <<< "$entry"
```
to:
```
IFS='|' read -r file_name tool_name <<< "$entry"
```
and update the two references to `dir_name` within that loop body (lines 82–83)
to `file_name`. Do not touch the `_KNOWN_AI_DIRS` loop above — `dir_name` is
correct there.
