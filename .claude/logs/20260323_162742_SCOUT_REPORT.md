## Suspected Root Cause Areas

- **Event ID threading discipline:** If a caller fails to capture emit_event() return value and pass it as caused_by to downstream events, causal chains will be disconnected. Every emission site must thread IDs correctly.
- **JSON generation in pure bash:** Special characters in report content (quotes, newlines, backslashes) will corrupt JS if not escaped. _json_escape() must be consistently applied to all string values.
- **File locking / atomic writes:** Dashboard data files regenerate frequently during runs. Must use atomic writes (tmpfile + mv) for consistency, especially if future versions add parallel stage execution.
- **Causal log size growth:** Verbose runs with many rework cycles can accumulate thousands of events. Must enforce CAUSAL_LOG_MAX_EVENTS cap with oldest-first eviction to keep memory/storage reasonable.
- **RUN_SUMMARY.json parsing:** Fallback grep/awk path for JSON must handle full format when python3 is unavailable or fails. Test both paths in CI.
- **Dashboard refresh interval:** Spinner's periodic refresh must use existing agent_monitor loop (no new background process). Must not block agent monitoring itself.
- **Cross-run archive queries:** Queries that scan multiple archived logs (50+ runs × 2000 events each = 100k+ lines) must use grep with targeted patterns first, not load everything into memory.
- **Manifest + milestone files:** Milestone state changes (advance, split) must emit events AND update manifest AND emit_dashboard_milestones() in correct order. Manifest update must be atomic.
- **Config key scope:** New keys (CAUSAL_LOG_*) are independent of DASHBOARD_ENABLED — causal log runs even when dashboard is off. Both are opt-in via config but default-on. Must be clear in documentation.
