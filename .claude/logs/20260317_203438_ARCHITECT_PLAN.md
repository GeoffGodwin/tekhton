## Simplification

### Split `lib/milestones.sh` into `milestones.sh` + `milestone_archival.sh`

**Observation**: `lib/milestones.sh` is 805 lines with three distinct areas of
responsibility: (1) milestone state machine (parsing, state file management,
acceptance checking — lines 1–414), (2) commit signatures and auto-advance
orchestration (lines 416–575), and (3) archival (_extract_milestone_block,
_get_initiative_name, _milestone_in_archive, archive_completed_milestone,
archive_all_completed_milestones — lines 576–805). The 300-line ceiling exists
to prevent exactly this multi-concern growth.

**Task (sr coder)**:

Extract the archival block (lines 576–805, ~229 lines) into a new
`lib/milestone_archival.sh`. The archival functions are self-contained: they
depend only on `log()`, `warn()`, `success()` from `common.sh` and
`MILESTONE_ARCHIVE_FILE` from config. No callers in `lib/milestones.sh` depend
on them at parse or acceptance time — they are invoked only from `tekhton.sh`
post-commit.

Specific actions:

1. Create `lib/milestone_archival.sh` with the standard `set -euo pipefail` header.
   Move these five functions verbatim into it:
   - `_extract_milestone_block`
   - `_get_initiative_name`
   - `_milestone_in_archive`
   - `archive_completed_milestone`
   - `archive_all_completed_milestones`

2. Remove those functions and their section comment (`# --- Milestone archival`)
   from `lib/milestones.sh`.

3. In `tekhton.sh`, add `source "${TEKHTON_HOME}/lib/milestone_archival.sh"` in
   the same block where `lib/milestones.sh` is sourced (the `--auto-advance`
   sourcing block and the main sourcing block). Both callers of archival functions
   (`archive_completed_milestone` at line ~1065 and `archive_all_completed_milestones`
   at startup) are in `tekhton.sh` so this sourcing change is the only wiring needed.

4. Update the `# Provides:` header in `lib/milestones.sh` to remove the archival
   function names. Add a corresponding `# Provides:` header in
   `lib/milestone_archival.sh`.

5. Update `ARCHITECTURE.md`: add `lib/milestone_archival.sh` to Layer 3 with a
   one-line description. Update the `lib/milestones.sh` description to remove
   archival mention.

**Acceptance**:
- `bash -n lib/milestones.sh` passes
- `bash -n lib/milestone_archival.sh` passes
- `shellcheck lib/milestones.sh lib/milestone_archival.sh` passes with zero warnings
- `bash tests/run_tests.sh` — all existing tests pass
- `wc -l lib/milestones.sh` ≤ 580 (archival block removed)
- `wc -l lib/milestone_archival.sh` ≤ 240

**Residual note**: After this split, `lib/milestones.sh` will be approximately
576 lines — still above the 300-line ceiling. The commit-signatures and
auto-advance block (lines 416–575, ~160 lines) could form a second extraction
(`lib/milestone_advance.sh`) in a follow-on run if the ceiling violation is
treated as blocking. That is a separate bounded task; it is not part of this run.

## Out of Scope

- **[2026-03-17] `tekhton.sh:1073` — `${COMMIT_MSG%%$'\n'*}` bash version concern**:
  Moving to out of scope. The `%%` operator with `$'\n'` is standard ANSI-C quoting
  in Bash 4+ parameter expansion. Tekhton's own non-negotiable rules mandate Bash 4+,
  so no cross-version issue exists within the supported range. The drift observation
  is explicitly speculative ("may behave differently", "worth watching if users
  report"). No user-reported truncation issues exist. Proposing a defensive
  substitution without evidence of breakage would be speculative refactoring, which
  is outside the architect mandate. The observation should be closed from the drift
  log without action.
