## Staleness Fixes

### Fix `grep -q` → `grep -qF` at `stages/cleanup.sh:272`

**File:** `stages/cleanup.sh`
**Function:** `_resolve_cleanup_by_file_changes`
**Line:** 272

The call `echo "$note_line" | grep -q "$basename_mod"` uses plain `-q` while the three
other `grep` calls in the same file (lines 128, 221, 241) all use `-qF` or `-qxF`.

The `-F` flag (fixed-string matching) is the correct choice here: `$basename_mod` is a
filesystem basename, not a regex pattern. Without `-F`, characters like `.` in filenames
(e.g., `cleanup.sh`) are treated as regex wildcards, which could produce false positives.

**Action:** Change line 272 from:

```bash
if echo "$note_line" | grep -q "$basename_mod" 2>/dev/null; then
```

to:

```bash
if echo "$note_line" | grep -qF "$basename_mod" 2>/dev/null; then
```

No other changes needed. One line, one file.
