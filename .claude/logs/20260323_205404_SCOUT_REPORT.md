# SCOUT_REPORT.md — Milestone 16: Autonomous Runtime Improvements

## Summary

Milestone 16 reforms the `--complete` / `--auto-advance` outer loop to enable indefinite productive execution. Three interconnected changes: (1) milestone success resets the pipeline attempt counter so progress resets the attempt budget, (2) quota-aware pause/resume system gracefully handles rate limits, and (3) increased split depth now that PM + security agents provide safety rails.

## Suspected Root Cause Areas

- **Agent-to-quota-to-orchestrate feedback loop:** Rate limit must surface from agent.sh → agent_retry.sh → quota.sh → orchestrate.sh. Integration points at: (1) error classification in agent.sh, (2) retry condition check in agent_retry.sh, (3) pause/resume orchestration in quota.sh, (4) pause state check in agent_monitor.sh.

- **Timeout coordination during pause:** AUTONOMOUS_TIMEOUT must be frozen (remaining time saved), not disabled, or long pauses allow indefinite post-resume execution. AGENT_ACTIVITY_TIMEOUT must be fully disabled (set to 0) during pause so monitoring doesn't trigger false timeouts on the quota retry probe.

- **Milestone success counter reset:** Must reset `pipeline_attempts` (not global attempt, but the outer loop's attempt counter) AFTER `mark_milestone_done()` returns 0. Must also reset on `split_milestone()` success. Failure to reset defeats the "indefinite productive runs" goal.

- **Progress detection for counter increment:** `_check_progress()` must distinguish "work was done but incomplete" (counter stays 0) from "no work done" (counter increments). Causal log provides richest signal; git-diff fallback still works for projects without log.

- **Activity monitor disable during pause:** agent_monitor.sh activity timeout is extended in several existing flows (MILESTONE_ACTIVITY_TIMEOUT_MULTIPLIER, etc.). During QUOTA_PAUSED, it must be fully disabled, not extended. The quota retry loop is NOT an agent invocation and should not be subject to activity monitoring.

- **Quota probe lightweight enforcement:** The quota check probe must be a single-turn `claude` call with a trivial prompt ("respond with OK"), NOT a full agent invocation. If it fails, the probe succeeded (quota refreshed). If it's rate-limited, quota hasn't refreshed yet.
