## Suspected Root Cause Areas

1. **Clarification Detection** (`stages/coder.sh` post-invocation):
   - Must parse `## Clarification Required` section from `CODER_SUMMARY.md`
   - Must extract and separate `[BLOCKING]` vs `[NON_BLOCKING]` items
   - Logic similar to `lib/notes.sh:extract_human_notes()` pattern (grep + sed)

2. **User Interaction & State Save** (`lib/clarify.sh:handle_clarifications()`):
   - Must read from `/dev/tty` for interactive prompting (platform-specific: Linux/macOS vs Windows Git Bash)
   - Must handle Ctrl+C and save state for resume
   - Must write `CLARIFICATIONS.md` with structured format for subsequent `render_prompt()` injection

3. **Replan Trigger & Menu** (`stages/review.sh` + `lib/clarify.sh:trigger_replan()`):
   - Must detect `REPLAN_REQUIRED` verdict from `REVIEWER_REPORT.md` (similar to blocker parsing in stages/review.sh:108-125)
   - Must present interactive menu: `[r]`, `[s]`, `[c]`, `[a]` (similar to plan resume confirmation pattern in lib/plan_state.sh)
   - Menu selection routes to different outcomes: replan → call `_call_planning_batch()`, split → TBD, continue → skip, abort → exit

4. **Context Re-Injection** (`prompts/coder.prompt.md` + `stages/coder.sh`):
   - Must add `{{IF:CLARIFICATIONS_CONTENT}}` block to coder prompt so clarification answers are available on resume
   - Export `CLARIFICATIONS_CONTENT` from stages/coder.sh before re-invoking agent
   - Similar pattern to existing `{{IF:PRIOR_REVIEWER_CONTEXT}}` blocks

5. **Replan Prompt & Approval** (`prompts/replan.prompt.md` + `stages/review.sh`):
   - Must generate `REPLAN_DELTA.md` showing proposed changes
   - Must display delta to user with approval menu: `[a]`, `[e]`, `[n]` (similar to plan review UI in lib/plan.sh)
   - If approved, merge changes into CLAUDE.md/DESIGN.md
