## Suspected Root Cause Areas

1. **Activity timeout false positives (line 308)**: The FIFO monitoring loop only tracks FIFO output (`_last_activity`). With `--output-format json`, agents may perform file I/O silently for minutes without streaming JSON text lines, triggering a false activity timeout before work is complete.

2. **Null-run declaration for completed work (lines 452-478)**: When an agent hits activity timeout (exit 124), it is automatically marked as null run regardless of whether files were actually modified. This discards completed work. The null-run heuristic needs a secondary signal: file-change detection via `git status --porcelain` and `git diff --stat`.

3. **Missing CODER_SUMMARY.md completion signal (line 254)**: The coder stage checks for CODER_SUMMARY.md existence, but the agent monitoring in `run_agent()` doesn't know to expect this file. If JSON output mode produces no streaming output, the activity timeout fires before CODER_SUMMARY.md is written.

4. **Lack of file-change activity detection (line 284)**: `_last_activity` is reset only when FIFO produces output. It should also reset when git working tree changes are detected. This requires periodic `git status --porcelain` checks during the read loop.

### Justification

- **Files to modify: 3** — lib/agent.sh (primary, ~100 lines), lib/config.sh (minor, config defaults), tests (new or extended test coverage)
- **Lines of change: 120** — Activity timeout detection augmented with git status polling (~20 lines), null-run detection enriched with git diff parsing (~30 lines), CODER_SUMMARY.md existence check in agent wrapper (~10 lines), helper functions for file-change detection (~20 lines), test cases (~40 lines)
- **Interconnected systems: medium** — Changes are isolated to lib/agent.sh except for config defaults in lib/config.sh. Activity timeout affects all agent invocations (coder, reviewer, tester, scout), so implementation must be careful not to slow down the read loop or cause regressions in existing null-run detection. Tests require new test cases that mock git state and file changes.
- **Coder turns: 35** — Moderate complexity. Must carefully integrate file-change detection into the FIFO read loop (performance-sensitive), parse git output safely, add new config variables, handle Windows/macOS git command variations, and maintain backward compatibility with existing activity timeout behavior.
- **Reviewer turns: 10** — Medium focus. Key review points: (1) Does git status polling impact performance? (2) Are git commands safe from injection? (3) Do file-change signals properly reset the activity timer? (4) Does CODER_SUMMARY.md check correctly detect agent completion?
- **Tester turns: 20** — Moderate test coverage. Must mock git state, simulate activity timeouts with file changes, test null-run detection with various exit codes + file-change combinations, verify CODER_SUMMARY.md existence check prevents false null runs, test on both text and JSON output modes.
