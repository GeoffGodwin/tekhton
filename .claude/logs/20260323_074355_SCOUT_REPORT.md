## Suspected Root Cause Areas

- **PageRank personalization vector** — `_rank_files()` in repo_map.py must blend three weighted signals. The history loading and keyword similarity calculation are the critical integration points; git recency requires subprocess calls to git log with timeout handling.

- **Cache versioning and migration** — tag_cache.py `load()` must detect version mismatch and invalidate gracefully. If version changes between runs, old cache entries must be discarded but the process must not crash.

- **Task history JSONL safety** — `record_task_file_association()` must be append-only (never read-modify-write) and handle concurrent writes safely. Pruning must use atomic tmpfile+mv pattern. History records may contain sensitive task descriptions — setup must warn users.

- **File deletion detection** — `prune_cache()` must use git ls-files or os.path.exists() to detect deleted files, then remove stale entries. Must handle git errors gracefully for non-git projects.

- **Metrics integration** — `record_run_metrics()` must capture cache statistics before cache.save(), and `get_indexer_stats()` must return consistent format for dashboard injection.
