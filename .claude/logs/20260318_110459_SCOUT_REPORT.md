## Suspected Root Cause Areas

- **Systemic library style issue**: `lib/milestone_split.sh` and `lib/milestone_archival.sh` lack `set -euo pipefail` preamble (lines 1–2), diverging from CLAUDE.md Rule 2 and established project convention. All existing `.sh` files include this. While they inherit the setting via `source` from `tekhton.sh`, this creates a fragile dependency — if ever sourced elsewhere (test harness, separate invocation), silent failures could result.

- **File size balloon**: `lib/milestones.sh` is now 282 lines, approaching the 300-line ceiling established in the code quality guidelines. Adding more functions (e.g., splitting orchestration, advanced acceptance checking) will exceed the limit. Future refactor should extract into a companion file (e.g., `lib/milestone_ops.sh`).

- **Latent duplication in split context**: Although `_switch_to_sub_milestone()` helper was recently extracted to reduce code duplication, the pattern of "split milestone → update state → re-scout" appears in three separate paths (pre-flight, null-run, turn-limit). Duplication is now encapsulated in the helper, but the broader orchestration (split → state → recursive call) remains as a conceptual pattern worth monitoring.
