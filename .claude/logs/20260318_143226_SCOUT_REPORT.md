## Key Symbols

Functions to implement (lib/errors.sh):
- `classify_error(exit_code, stderr_file, last_output_file)` — Parses exit code + stderr/output to return `CATEGORY|SUBCATEGORY|TRANSIENT|MESSAGE`
- `is_transient(category, subcategory)` — Returns 0 if transient (retry safe), 1 if permanent (requires fix)
- `suggest_recovery(category, subcategory, context)` — Returns human-readable recovery text for each error type
- `redact_sensitive(text)` — Strips API keys, auth tokens, preserves Anthropic request IDs

Error taxonomy to encode:
- `UPSTREAM` category with subcategories: `api_500`, `api_rate_limit`, `api_overloaded`, `api_auth`, `api_timeout`, `api_unknown`
- `ENVIRONMENT` category with subcategories: `disk_full`, `network`, `missing_dep`, `permissions`, `oom`, `env_unknown`
- `AGENT_SCOPE` category with subcategories: `null_run`, `max_turns`, `activity_timeout`, `no_summary`, `scope_unknown`
- `PIPELINE` category with subcategories: `state_corrupt`, `config_error`, `missing_file`, `template_error`, `internal`

## Suspected Root Cause Areas

- **classify_error() detection logic** — Must parse Claude CLI JSON error output format reliably. Will use grep patterns on stderr + last output to detect: `"type":"server_error"`, `"type":"rate_limit_error"`, `"type":"overloaded_error"`, HTTP status codes (429, 500, 502, 503, 529), auth failures
- **Transience taxonomy** — Must correctly classify which errors are safe to retry (all UPSTREAM, ENVIRONMENT/network, ENVIRONMENT/oom) vs permanent (all AGENT_SCOPE, PIPELINE). Accuracy critical for Milestone 12.2 integration
- **Redaction patterns** — Must identify and strip: `x-api-key: *`, `Authorization: *`, `sk-ant-*`, `ANTHROPIC_API_KEY=*`. Must NOT redact Anthropic request IDs (`req_*`). Conservative over-redaction preferred
- **Recovery suggestion accuracy** — Each subcategory must have actionable, specific recovery text. Generic suggestions defeat the purpose
- **Fallback to *_unknown** — Unrecognized error patterns must gracefully fall back to `*_unknown` subcategories, never error. This is a pure library with no side effects

**Interconnectedness**: Low. Milestone 12.1 is pure library with no integration into agent.sh, stages, or hooks. All functions are independently testable in isolation. - Error taxonomy requires complete mapping of all error categories + subcategories (4 categories × 5-6 subcategories each = ~24 distinct error states)
- Classification logic must handle both JSON error format detection (grep patterns on stdout/stderr) and exit code inference
- Transience logic is simple once taxonomy is clear (lookup table)
- Recovery suggestion mapping (24 unique messages) is straightforward but must be specific
- Redaction requires regex patterns for API key formats and conservative heuristics
- No control flow complexity, no state management, no side effects
