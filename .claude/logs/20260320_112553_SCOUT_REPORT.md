## Suspected Root Cause Areas

- **New library creation**: `lib/detect.sh` is entirely new and must follow Tekhton's bash conventions (set -euo pipefail, shellcheck clean, portable)
- **Manifest file detection**: Must safely parse `package.json`, `Cargo.toml`, `pyproject.toml`, `go.mod`, `Gemfile`, `build.gradle`, `pom.xml` without executing arbitrary code. Pattern exists in `lib/config.sh` for safe parsing
- **Language detection via file extensions and shebangs**: Must handle edge cases (vendored code, monorepos, mixed stacks). Exclusion list must match the one used by `_generate_codebase_summary()` if it exists
- **Framework detection from dependencies**: Must parse manifest dependency lists and map well-known packages (express, react, flask, etc.) to framework names. Case-sensitive matching required
- **Command detection from manifest scripts**: Must extract test/build/lint commands from `package.json` scripts, `Cargo.toml` [build], `pyproject.toml` [tool.pytest], etc. Fallback to well-known tool detection (npm, cargo, pytest binaries)
- **Entry point detection**: Must find primary executable(s) ‚Äî `main.py`, `index.ts`, `src/main.rs`, `cmd/*/main.go`, `lib/main.dart`, `Program.cs`, `App.java`, `Makefile`, `docker-compose.yml`. Order matters for priority
- **Project type classification**: Logic to map detected language + framework + entry points to one of 7 templates. Heuristics: Node.js + React ‚Üí web-app, Rust + CLI args ‚Üí cli-tool, etc.
