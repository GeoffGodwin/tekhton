## Suspected Root Cause Areas

1. **lib/plan.sh file size** — Currently 290 lines; Milestone 6 (state persistence + config) will push over 300-line threshold. Need to pre-plan module split (e.g., `lib/plan_completeness.sh` for completeness logic)

2. **run_plan_completeness_loop invalid-choice handler** — lines 257-261 decrement `pass_num` then continue loop, which re-increments at line 219, causing second "Completeness Check — Pass N" header for same logical pass. Inner choice re-prompt loop would fix.

3. **Missing `set -euo pipefail` headers** — lib/plan.sh and stages/plan_interview.sh inherit from tekhton.sh at runtime but CLAUDE.md rule #2 requires explicit headers in all scripts for standalone testability

4. **Variable shadowing in tests** — test_plan_completeness.sh line 14 `TMPDIR=$(mktemp -d)` shadows system environment variable used by many programs; rename to `TEST_TMPDIR`

5. **Stdin handling** — select_project_type() uses `read -r choice` with no `/dev/tty` fallback; blocks silently when invoked with piped stdin (problem for CI/scripted testing)

6. **Missing startup warning** — run_plan_interview() uses `--dangerously-skip-permissions` but doesn't warn user about interview agent write access to DESIGN.md

7. **ARCHITECTURE.md outdated** — Layer 1 description does not reflect that `--plan` block now sources `agent.sh` (lines 171-173 in tekhton.sh) in addition to the documented libraries
