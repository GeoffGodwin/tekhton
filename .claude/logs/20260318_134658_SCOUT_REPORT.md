## Suspected Root Cause Areas

1. **Environment bloat:** Lines 146-149 in `lib/milestone_split.sh` export potentially large strings (`MILESTONE_DEFINITION` can be hundreds of lines) that persist for all child processes in milestone mode, not just for `render_prompt()` and `_call_planning_batch()` calls

2. **File tracking gap:** Coder prompt instructs manual "Files Modified" tracking (coder.prompt.md:65-66), and completion gate doesn't validate this list (lib/gates.sh:111-135), so omissions go undetected. Affects `lib/milestone_archival.sh` specifically across three recent cycles

3. **Em-dash regex patterns:** Portable regex delimiters issue in `lib/milestone_archival.sh` at four locations:
   - Line 35: bash regex `[:.\ -]|—` in `_extract_milestone_block()`
   - Line 80: bash regex `[:.\ -]|—` in `_get_milestone_initiative()`
   - Line 163: awk regex `[:. -]|—` in `_replace_milestone_block()` (within awk block)
   - Line 225: awk regex `[:. -]|—` in `_archive_milestone_to_file()` (within awk block)

   The em-dash (U+2014) outside the bracket is not portable; some shells/regex engines mishandle this pattern

**Issue 3 — Em-dash Regex Portability (1 file, 4 locations):**
- `lib/milestone_archival.sh` lines 35, 80, 163, 225: Replace em-dash patterns with portable alternatives
- Example fix: `[:.\ -]|—` → `[.:\ -]|[—]` or use character class escape instead: `[:.\ -]|[$(printf '\u2014')]`
- OR: Use separate alternation without character class: `[:.]|[ -]|—` simplified to ASCII-safe matching
- Estimated: 4-6 lines (one per occurrence, mostly straightforward substitution)
