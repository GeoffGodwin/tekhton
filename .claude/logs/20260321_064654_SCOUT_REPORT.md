## Suspected Root Cause Areas

- **Flag wiring in tekhton.sh** (line 192-208): Must add --plan-from-index early exit block with proper library sourcing (plan.sh for _call_planning_batch, context.sh for budget checking)
- **Context budget application** (lib/context_compiler.sh): PROJECT_INDEX.md + detection report + sampled file content can exceed context window on large projects; compression strategy must truncate file inventory first, then sample headings-only
- **Completeness checking on synthesized DESIGN.md** (lib/plan_completeness.sh): Agent may produce shallow sections; follow-up interview loop needed (reuse `run_plan_completeness_loop()` pattern from plan.sh:317)
- **Prompt quality for brownfield synthesis** (new prompts): Must emphasize "document what EXISTS in the codebase" not "prescribe what should be built"; DESIGN.md synthesis agent must avoid hallucinating patterns
- **CLAUDE.md milestone generation for brownfield** (init_synthesize_claude.prompt.md): Brownfield milestones are "refactor X", "add tests for Y", "document Z" not "build feature W"; prompt must make this distinction explicit
- **State persistence across phases** (optional PLAN_STATE_FILE usage): If synthesis is interrupted, resume should re-run from last failed phase; state file pattern from plan_state.sh can be adapted
