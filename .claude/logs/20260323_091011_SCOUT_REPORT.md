## Suspected Root Cause Areas

1. **State Resume Chain** (`lib/state.sh` line unknown) — Current supports only "coder", "review", "test"; missing "security". Attempting `--start-at security` will fail validation.

2. **Config Validation** (`lib/config.sh` end-of-file) — No validation for SECURITY_UNFIXABLE_POLICY enum or SECURITY_BLOCK_SEVERITY enum. Invalid values will silently use defaults.

3. **Template Variable Binding** (`lib/prompts.sh`) — SECURITY_FINDINGS_BLOCK and SECURITY_FIXES_BLOCK are set by security.sh but not registered in prompts.sh render_prompt(), so {{SECURITY_FINDINGS_BLOCK}} in reviewer prompt will remain unexpanded.

4. **Prompt Integration** (`prompts/reviewer.prompt.md` and `prompts/tester.prompt.md`) — No conditional blocks to inject security context. Reviewer and tester will be unaware security fixes were applied.

5. **Archive Missing** (`lib/hooks.sh` or `lib/finalize.sh`) — SECURITY_REPORT.md and SECURITY_NOTES.md are not copied to .claude/logs archive with timestamp. Historical security findings are lost between runs.

6. **Stage Numbering** (multiple files) — Headers say "Stage 3 / 4" but pipeline is being called as 3-stage. Confuses users and breaks progress tracking.
