## Implementation Strategy

**Phase 1: Completeness Detection Function**
- Add `check_design_completeness()` to lib/plan.sh
- Parse DESIGN.md for section headers (grep `^##`)
- Validate required sections from template (cross-reference with PLAN_TEMPLATE_FILE)
- Check each section for:
  - Empty content (only header, no text below)
  - Guidance comments still present (`<!-- ... -->`)
  - Placeholder-only content (e.g., just "TBD", "TK")
- Return structured output: list of incomplete section names + descriptions

**Phase 2: Follow-Up Interview Loop**
- Add loop in stages/plan_interview.sh or lib/plan.sh
- Call completeness checker after initial interview
- If incomplete sections found, prompt user to continue
- Re-launch claude in conversational mode targeting only incomplete sections
- Loop until all required sections pass or user opts to skip

**Phase 3: Follow-Up Prompt Variant**
- Either enhance plan_interview.prompt.md to handle follow-up context
- Or create new prompts/plan_interview_followup.prompt.md
- Follow-up prompt explains which sections are incomplete and asks user to fill them

**Phase 4: Tests**
- Create tests/test_plan_completeness.sh
- Test detection of empty sections, guidance comments, placeholders
- Test follow-up loop behavior
- Test determinism (same DESIGN.md always produces same result)
