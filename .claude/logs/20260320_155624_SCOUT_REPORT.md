## Suspected Root Cause Areas
- Manifest parsing: Multiple language formats (package.json, Cargo.toml, pyproject.toml, go.mod, Gemfile, build.gradle, pom.xml) all have slightly different formats. Reuse `_extract_json_keys()` pattern from detect_commands.sh but extend to TOML/YAML/XML parsing
- File inventory generation: Need to efficiently count files and lines without excessive `wc` calls. Use `find ... -exec wc -l {} +` batching pattern to avoid per-file overhead
- Budget allocation: Complex logic to distribute fixed percentages (tree 10%, inventory 15%, dependencies 10%, config 5%, tests 5%, sampled files 55%) across sections with surplus redistribution. Must be conservative to avoid context window overflow
- Binary file detection: When sampling files, must detect and skip binary files (use `file --mime` or check for null bytes in first 512 bytes)
- Monorepo handling: Large repos may have sub-projects (packages/, apps/). Must detect and cap at 5 sub-projects to prevent budget explosion
- Symlink handling: Must not follow symlinks outside project tree (use `find -not -L` or explicit symlink detection)
