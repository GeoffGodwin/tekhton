### Core Functionality Implemented

**detect_languages()** (lib/detect.sh:21-93)
- Scans manifest files (package.json, Cargo.toml, go.mod, pyproject.toml, Gemfile, pom.xml, .csproj, etc.)
- Counts source files by extension at top 2 directory levels
- Assigns confidence: "high" (manifest + sources), "medium" (manifest OR 5+ sources), "low" (few sources)
- Supports 15+ languages: JavaScript, TypeScript, Python, Rust, Go, Java, Kotlin, C#, Ruby, PHP, Dart, Swift, Elixir, Haskell, Shell
- Respects .gitignore and exclusion patterns (node_modules, .git, __pycache__, vendor, target, etc.)

**detect_frameworks()** (lib/detect.sh:171-247)
- Extracts dependencies from package.json (Node.js), pyproject.toml/requirements.txt (Python), Gemfile (Ruby), build.gradle/pom.xml (Java), Package.swift (Swift), Cargo.toml (Rust), go.mod (Go)
- Detects frameworks: React, Next.js, Vue, Angular, Svelte, Express, Fastify, Django, Flask, FastAPI, Rails, Spring Boot, ASP.NET, Flutter, Gin, Actix, Axum, and more
- Returns FRAMEWORK|LANG|EVIDENCE format for downstream use

**detect_commands()** (lib/detect_commands.sh:16-136)
- Extracts test/build/analyze commands from package.json scripts (highest priority)
- Fallback to language conventions: `cargo test` (Rust), `pytest` (Python), `go test` (Go), `make test` (Makefile), etc.
- Supports 12+ language/build-system combinations
- Returns CMD_TYPE|COMMAND|SOURCE|CONFIDENCE format
- Detects both explicit (package.json) and inferred (tool presence) commands

**detect_entry_points()** (lib/detect_commands.sh:143-172)
- Identifies canonical entry points: main.py, app.py, index.ts, src/main.rs, cmd/*/main.go, lib/main.dart, Program.cs, App.java, Makefile, docker-compose.yml, etc.
- Special handling for Go `cmd/` pattern (finds cmd/*/main.go)
- Returns file paths relative to project root

**detect_project_type()** (lib/detect_commands.sh:179-263)
- Classification logic: mobile-app (Flutter/SwiftUI), web-app (React/Next/pages dir), api-service (Express/Django/FastAPI), web-game (phaser/pixi/three/babylon), cli-tool (Rust CLI deps, Go cmd pattern), library (no entry points + manifest), custom (fallback)
- Uses detect_languages(), detect_frameworks(), detect_entry_points() as inputs
- Returns one of: web-app, web-game, cli-tool, api-service, mobile-app, library, custom

**format_detection_report()** (lib/detect_report.sh:15-74)
- Renders all detection results as structured markdown tables
- Sections: Tech Stack Detection Report header, Languages table, Frameworks table, Detected Commands table, Entry Points list, Project Type annotation
- Handles empty detection gracefully ("(none detected)")
- Output suitable for inclusion in PROJECT_INDEX.md or agent prompts
