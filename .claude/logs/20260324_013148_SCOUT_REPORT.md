- `tests/test_docs_site.sh` validates file existence and nav structure but does not run `mkdocs build` or `mkdocs serve` to test actual rendering

- No verification that all internal navigation links are correct (only that files referenced in mkdocs.yml exist)

**Content Quality:**
- Docs are scaffolded (exist, have headings) but may lack sufficient depth to satisfy acceptance criterion: "written for idea person who isn't a senior dev, not just engineers"
- Some getting-started docs may reference features (Watchtower, health scoring) that aren't fully explained inline
