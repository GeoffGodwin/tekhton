## Suspected Root Cause Areas

- **Interactive prompt sourcing**: `lib/prompts_interactive.sh` must be sourced by `lib/init.sh` before calling `prompt_confirm()` and `prompt_choice()`. Already done at line 19 of init.sh.
- **Detection function availability**: Detection functions must be available when `run_smart_init()` is called. Already sourced in tekhton.sh early-exit block before init.sh.
- **Crawler budget allocation**: `crawl_project()` takes a budget in chars (default 120000). Budget must be sufficient for PROJECT_INDEX.md. Already hardcoded to 120000 at line 76 of init.sh.
- **Agent role addenda append order**: Addenda must be appended to templates without overwriting security directives. Already done via `cat >> "$target"` pattern at line 242 of init.sh.
- **Git tracking vs file walk fallback**: When `git ls-files` fails, code falls back to `find` with exclusion patterns. Already implemented at lines 192-203 of init.sh.
- **Non-interactive mode fallback**: Tests set `TEKHTON_NON_INTERACTIVE=true` to avoid prompting. Prompts return defaults when /dev/tty unavailable. Already implemented in prompts_interactive.sh.
