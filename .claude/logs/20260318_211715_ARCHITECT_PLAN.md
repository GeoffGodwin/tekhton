## Staleness Fixes

**1. `lib/drift.sh` header missing cross-file dependency note**

`drift.sh` calls `append_nonblocking_notes()` and `_resolve_addressed_nonblocking_notes()`
at lines 393 and 396 inside `process_drift_artifacts()`. These functions are defined in
`drift_cleanup.sh`, not in `drift.sh`. The `drift_cleanup.sh` header already documents
the reverse dependency ("Uses: `_awk_join_bullets()` from drift.sh"), but `drift.sh`'s
header has no corresponding note that `drift_cleanup.sh` must be sourced after it for
`process_drift_artifacts()` to work.

**Action**: Add one line to the `drift.sh` header comment block:

```
# Depends on: drift_cleanup.sh sourced afterward (provides append_nonblocking_notes,
#             _resolve_addressed_nonblocking_notes)
```

Place it directly below the existing `# Expects:` block (after line 8), before the
closing `# =====` line.

File: `lib/drift.sh`, header (lines 1–9).

## Simplification

**1. `lib/agent_monitor.sh` — timed and blocking FIFO reader branches have diverged**

The FIFO reader loop (lines 135–226) has two branches:

- **Timed branch** (lines 136–211): carries ring buffer accumulation, real-time API
  error detection (`_stream_api_error`, `_stream_api_type`), and activity file scanning.
- **Blocking branch** (lines 212–225): stripped-down copy — only echoes the line and
  does text extraction. No ring buffer, no API error detection. Any future feature
  added to the timed branch must be manually mirrored.

**Action**: Extract a `_process_fifo_line(line)` helper function that encapsulates
the shared per-line work: (1) echo to fd 3, (2) update `_last_line`, (3) ring buffer
store (`_rb[$((_rb_idx % _rb_size))]="$line"; _rb_idx=$((_rb_idx + 1))`),
(4) API error pattern detection via the existing `case` block, (5) JSON text
extraction via `python3`. Both the timed and blocking branches then call this helper
instead of duplicating the logic.

The ring buffer array (`_rb[]`, `_rb_idx`) must remain in the subshell scope where
both branches live — the helper should take the line as `$1` and modify the subshell
globals directly (not via `local`). The existing comment at lines 234–240 explaining
why `local` is not used in the compound group remains accurate and should be preserved.

File: `lib/agent_monitor.sh`, lines 135–225.
