## Simplification

### 1. Remove invalid `local` declarations from compound group in `lib/agent_monitor.sh`

**File:** `lib/agent_monitor.sh`, lines 237, 239, 243

**Observation:** The ring buffer dump block (lines 236–249) uses `local` inside a
`{...}` compound group that lives inside a subshell `(...)` but NOT inside a function.
In bash, `local` is only valid inside a function body. When called outside a function,
bash returns exit code 1 and prints an error to stderr. With `set -euo pipefail` active
inside the subshell, the first `local` call (line 237) causes the compound group to exit
immediately — before it writes a single line to `agent_last_output.txt`. The
`2>/dev/null || true` wrapper suppresses the error and prevents pipeline failure, but
the ring buffer is never dumped. Error classification (Milestone 12.2) is silently broken
on every run as a result.

**Fix:** In the compound group at lines 236–249, replace the three `local` declarations
with plain variable assignments. The enclosing subshell `(...)` already provides scope
isolation — `local` is both invalid and unnecessary here.

Lines to change:
- Line 237: `local _rb_total=${#_rb[@]}` → `_rb_total=${#_rb[@]}`
- Line 239: `local _rb_start=0` → `_rb_start=0`
- Line 243: `local _j=0` → `_j=0`

No other changes needed. The ring buffer declaration (`declare -a _rb=()`) and the
modular index logic are correct; only the three `local` keywords in the dump block
need to be removed.

**Routing:** Sr coder (bash scoping semantics require understanding of subshell vs
function scope; the fix is three characters removed per line but the diagnosis is
non-obvious).
