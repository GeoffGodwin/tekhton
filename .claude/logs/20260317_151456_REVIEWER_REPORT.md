## Non-Blocking Notes
- `stages/coder.sh`: The post-clarification coder re-run (lines 318–327) does not update `ACTUAL_CODER_TURNS` after completion. If a blocking clarification triggers a second coder pass, the reviewer/tester turn calibration will use the original coder's turn count, not the re-run's. The spec only covers the primary coder run, so this is technically out-of-scope, but worth noting as a future gap.
- `lib/turns.sh` line 163–165: `grep -o '[0-9]* insertion' | tr -dc '0-9'` will produce an empty string (not "0") when the grep finds no match and the `||` fallback fires — but only after `tr` processes the empty input. In practice `tr` exits 0 on empty input so `|| echo "0"` never fires when `grep` matches but emits no digits. Functionally correct but slightly confusing — a comment would help.
- `lib/turns.sh` line 156: The awk pattern `(Modified|created or modified)` uses ERE alternation in a `/pattern/` context. This works in gawk (default on Linux/WSL) but is not POSIX-portable. Not an issue for the target environment, but worth a comment.

## Coverage Gaps
- No test for the case where `CODER_SUMMARY.md` is absent when `estimate_post_coder_turns` is called (e.g., via `--start-at review` with no summary present). The function defaults `files_modified` to 0 via the `[ -f "CODER_SUMMARY.md" ]` guard, which is correct, but there is no explicit test asserting `files_modified=0` in that path.
