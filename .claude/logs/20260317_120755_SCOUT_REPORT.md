## Suspected Root Cause Areas

- **Integration point 1:** After main reviewer completes with APPROVED/APPROVED_WITH_NOTES verdict in stages/review.sh (around line 240), need conditional invocation of `run_specialist_reviews()`
- **Integration point 2:** If specialist findings include [BLOCKER], must trigger rework routing (similar to reviewer blockers, around line 120-180 in stages/review.sh)
- **Integration point 3:** If specialist findings include [NOTE], append to NON_BLOCKING_LOG.md (use lib/notes.sh functions or direct append)
- **Config integration:** lib/config.sh defaults section needs ~20 lines for three built-in specialists + custom specialist support
- **Custom specialist support:** Pipeline must iterate over all `SPECIALIST_CUSTOM_*_ENABLED` keys and invoke each if true
- **Permission scope:** Specialists are readers (Read, Glob, Grep, Write) — no Bash/Edit needed. Reuse AGENT_TOOLS_REVIEWER or similar
