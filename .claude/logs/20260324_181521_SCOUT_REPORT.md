# Scout Report — Milestone 22: Init UX Overhaul

## Relevant Files

### Primary Implementation Files
- `lib/init_report.sh` — Post-init report generator (350 lines)
  - `emit_init_summary()` — prints focused, color-coded terminal summary with detected values, needs-attention items, health score, next steps
  - `emit_init_report_file()` — writes persistent INIT_REPORT.md with complete detection results, config decisions, and attention items
  - Helper functions: `_emit_summary_command()`, `_is_watchtower_enabled()`, report section helpers

- `lib/init_config_sections.sh` — Sectioned config file generator (304 lines)
  - `generate_sectioned_config()` — Main entry point producing complete pipeline.conf with 7 clearly separated sections
  - `_emit_section_essential()` — Section 1: Essential config (lines 1-20) with TEST_CMD, ANALYZE_CMD, BUILD_CHECK_CMD, ARCHITECTURE_FILE, PROJECT_RULES_FILE, DESIGN_FILE
  - `_emit_section_models_turns()` — Section 2: Models and turn limits
  - `_emit_section_pipeline()` — Section 3: Pipeline behavior (advanced settings)
  - `_emit_section_security()` — Section 4: Security agent configuration
  - `_emit_section_features()` — Section 5: Optional features (repo map, serena, dashboard, cleanup, etc)
  - `_emit_section_quotas()` — Section 6: Quotas & autonomy (autonomy mode controls)
  - `_emit_section_paths()` — Section 7: Paths & files (role files, log directories)
  - `_emit_section_workspace()` — Conditional: Monorepo/multi-service structure
  - `_emit_verified_line()` — Appends VERIFY/SUGGESTION markers for low-confidence detections

## Suspected Root Cause Areas

1. **Incomplete function implementations in init_report.sh or init_config_sections.sh**
   - Helper functions like `_report_*()` or `_emit_section_*()` may have incomplete implementations
   - May need to verify all color variables (GREEN, BOLD, YELLOW, CYAN, RED, NC) are available from common.sh

2. **Missing helper function: compute_health_score()**
   - `emit_init_summary()` calls `compute_health_score()` (line 107) with graceful skip
   - This is from M15 (Project Health Scoring) — may not be available yet
   - Current code gracefully handles this, but verify the conditional

3. **Dashboard integration**
   - `emit_dashboard_init()` is called but requires dashboard directory structure to exist
   - May need to ensure `.claude/dashboard/data/` directory is created during init
   - May need to verify _write_js_file() is available from dashboard_emitters.sh

4. **Config sectioning edge cases**
   - `_emit_section_workspace()` is conditional on monorepo/multi-service detection
   - May not handle single-repo projects correctly (should be a no-op)
   - May need to verify _INIT_WORKSPACES, _INIT_SERVICES, _INIT_WORKSPACE_SCOPE exports

5. **VERIFY marker logic**
   - `_emit_verified_line()` in init_config_sections.sh handles high/medium/low confidence
   - Low confidence puts comments on separate lines — verify this doesn't break sourcing
   - Medium confidence adds `# VERIFY:` comment inline

6. **Report metadata HTML comments**
   - `emit_init_report_file()` uses HTML comments for machine-parseable metadata
   - `emit_dashboard_init()` parses these comments with specific format: `<!-- init-report-meta ... -->`
   - Must ensure comment format matches parser expectations exactly
