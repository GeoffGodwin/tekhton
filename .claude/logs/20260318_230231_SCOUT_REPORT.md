# Scout Report: Milestone 13.2.1 — Core Retry Envelope in run_agent()

## Relevant Files

- lib/agent.sh — Main agent invocation wrapper (253 lines). Calls _run_with_retry() from agent_helpers.sh at lines 121-123. Receives results via globals (_RWR_EXIT, _RWR_TURNS, _RWR_WAS_ACTIVITY_TIMEOUT, AGENT_ERROR_*, LAST_AGENT_RETRY_COUNT). Processes retry-augmented results and appends agent summary (lines 131-252).
- lib/agent_helpers.sh — Retry envelope implementation (330 lines). Contains _run_with_retry() (lines 18-85), _classify_agent_exit() (lines 87-127), and _should_retry_transient() (lines 129-195). Fully implements exponential backoff, subcategory-specific delays, retry-after parsing, and monitoring state reset.
- lib/agent_monitor.sh — Agent process lifecycle and FIFO monitoring. Contains _reset_monitoring_state() (lines 333+) that cleans up FIFO, temp files, API error flags, and activity timestamps between retry attempts.
- lib/common.sh — Shared utilities. Contains report_retry() function (lines 116+) for formatted retry notices with Unicode/ASCII fallback.
- lib/errors.sh — Error classification engine (250+ lines). Contains classify_error() that produces structured error records (CATEGORY|SUBCATEGORY|TRANSIENT|MESSAGE) and is_transient() for checking if errors should be retried.
- lib/config.sh — Configuration loader. Lines 252-255 set defaults for TRANSIENT_RETRY_ENABLED, MAX_TRANSIENT_RETRIES, TRANSIENT_RETRY_BASE_DELAY, TRANSIENT_RETRY_MAX_DELAY. Lines 313-315 apply config value clamping with hard upper bounds.
- lib/errors_helpers.sh — Error helper functions. Contains suggest_recovery() and redact_sensitive() used by agent.sh for error reporting.

## Key Symbols

- _run_with_retry() — lib/agent_helpers.sh (line 18). Core retry envelope. Wraps _invoke_and_monitor in loop with error classification, transient check, exponential backoff, and monitoring state reset.
- _should_retry_transient() — lib/agent_helpers.sh (line 129). Decision point for retry. Checks TRANSIENT_RETRY_ENABLED, AGENT_ERROR_TRANSIENT flag, attempt count, calculates exponential backoff, applies subcategory-specific minimums (60s for rate_limit/overloaded, 15s for oom).
- _classify_agent_exit() — lib/agent_helpers.sh (line 87). Calls classify_error() and sets AGENT_ERROR_* globals.
- run_agent() — lib/agent.sh (line 52). Entry point. Calls _run_with_retry() and consumes all result globals, applies null-run detection heuristics, appends structured summary to log.
- classify_error() — lib/errors.sh (line 55). Takes exit code, stderr, last output files and produces CATEGORY|SUBCATEGORY|TRANSIENT|MESSAGE.
- is_transient() — lib/errors.sh. Determines if error should be retried (all UPSTREAM transient, some ENVIRONMENT transient, all AGENT_SCOPE/PIPELINE permanent).
- report_retry() — lib/common.sh (line 118). Formatted retry notice with attempt number, delay, category, and Unicode/ASCII fallback.
- _reset_monitoring_state() — lib/agent_monitor.sh (line 335). Kills lingering FIFO reader, removes temp files (stderr, output ring buffer, exit/turns files), resets API error flags.

## Suspected Root Cause Areas

- Retry logic placement: The original plan called for a `_retry_agent()` wrapper inside `run_agent()`, but implementation has delegated this to `_run_with_retry()` in agent_helpers.sh (called from run_agent). This is architecturally sound (keeps agent.sh under 300 lines) but differs from CLAUDE.md wording.
- Verification that _reset_monitoring_state() properly kills background FIFO reader subshell before removing FIFO file. If subshell persists, retry attempt 2 will conflict with stale FIFO from attempt 1.
- Verification that AGENT_ERROR_* variables are correctly reset at the top of each retry loop iteration (lines 39-43 in agent_helpers.sh). If not reset, retry N may classify based on error from retry N-1.
- Verification that the retry loop terminates after MAX_TRANSIENT_RETRIES exhausted and falls through to existing error path (currently handled correctly in _should_retry_transient return 1 at line 144).
- Verification that metrics integration (LAST_AGENT_RETRY_COUNT tracking) properly increments at line 79 and is exported to run_agent() caller.
- Testing of rate_limit (429) retry-after header parsing (lines 165-175 in _should_retry_transient). Pattern matching for quoted/unquoted retry-after values may be fragile.
- Verification that process group cleanup happens (mentioned in CLAUDE.md as "Consider process group cleanup" at line 13 of spec, but grep shows no `kill -- -$PID` in current code).
