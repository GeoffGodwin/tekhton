- Issue: If `${text}` contains literal backslash sequences (e.g., from a note with `\n` or `\t` in it), sed interprets them as escape codes rather than literal characters
- Fix: Use `printf %s` to quote the text before passing to sed, or switch to a here-doc append that doesn't interpret escapes
