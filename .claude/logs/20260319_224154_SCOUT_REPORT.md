# Scout Report: Milestone 15.4.1
## Single-Note Utility Functions in lib/notes.sh

### Task Summary
Implement five single-note utility functions in `lib/notes.sh` that enable the `--human` workflow:
- `pick_next_note(tag_filter)` — Find next unchecked note by priority
- `claim_single_note(note_line)` — Mark ONE note as [~] (in-progress)
- `resolve_single_note(note_line, exit_code)` — Mark [~] note as [x] or reset to [ ]
- `extract_note_text(note_line)` — Strip checkbox prefix from note
- `count_unchecked_notes(tag_filter)` — Count remaining [ ] items

## Suspected Root Cause Areas

1. **Section boundary detection in pick_next_note()** — Must correctly identify section boundaries (## Bugs, ## Features, ## Polish) to prioritize bug fixes over features. Watch for off-by-one errors in section scanning.

2. **Sed regex escaping in claim_single_note() and resolve_single_note()** — Note text may contain brackets, parentheses, dots, or other regex metacharacters. The existing code in resolve_human_notes() (lines 132) demonstrates the escaping pattern using `sed 's/[.[\*^$()+?{|/]/\\&/g'` but the new functions must apply this consistently.

3. **Tmpfile handling in claim_single_note()** — Must follow the same tmpfile+mv pattern used by drift_cleanup.sh for atomicity. Cannot use sed -i for portability (BSD/GNU differences).

4. **Extract_note_text() edge cases** — Must handle variant checkbox formats: `[ ]`, `[~]`, `[x]` plus optional spaces.

5. **Empty file and missing section handling** — All functions must gracefully handle missing HUMAN_NOTES.md and return appropriate defaults (0, empty string) without erroring.
