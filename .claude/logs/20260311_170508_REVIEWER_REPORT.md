## Non-Blocking Notes
- `tests/test_agent_fifo_invocation.sh` lines 125, 155, 185, 217, 238, 241: uses `PATH="${MOCK_BIN}:${PATH}" run_agent ...` as if it creates a temporary environment for the function call, but bash only applies the temporary-assignment semantics to external commands — for shell functions the variable is modified in the current shell and persists. The test works correctly because `MOCK_BIN` is always prepended before each call anyway, but the technique is misleading. Simpler and clearer: export `PATH="${MOCK_BIN}:${PATH}"` once at the top of the test file.
- `tests/test_agent_fifo_invocation.sh` line 17: uses `TMPDIR` as the test temp dir variable, which shadows the conventional `$TMPDIR` system environment variable. Renaming to `TEST_TMP` or `_TMPDIR` would avoid the shadowing. Not a correctness issue in practice since no mock script uses `$TMPDIR`, but worth a future-clean.
- The activity-timeout null run warning at Phase 6 outputs "agent timed out after 0s" because `AGENT_TIMEOUT=0` (outer timeout disabled) leaves `$_timeout` as the literal string `0` in the warn message. Pre-existing behavior — not introduced by this change — but confusing.
