## What Was Implemented

Milestone 5: Milestone Review UI + File Output — the interactive review/approval step after CLAUDE.md generation.

- **`_extract_project_name()`** — Parses project name from CLAUDE.md's first heading, falls back to directory name
- **`_extract_milestones()`** — Extracts "## Milestone N:" or "### Milestone N:" headings from CLAUDE.md
- **`_display_milestone_summary()`** — Renders the formatted milestone summary screen with project name, milestone count, numbered milestone list, and action menu
- **`_print_next_steps()`** — Prints post-acceptance instructions (review files, run --init, start milestones)
- **`run_plan_review()`** — Interactive review loop with four actions:
  - `[y]` Accept — confirms files are written, prints next-steps instructions
  - `[e]` Edit — opens CLAUDE.md in `$EDITOR` (default: nano), refreshes summary after editor closes
  - `[r]` Re-generate — re-runs `run_plan_generate()` with the same DESIGN.md, then loops back to review
  - `[n]` Abort — warns user that files are preserved, returns non-zero
- **Wired into `run_plan()`** as Step 5 after generation (replaced "Future milestones" comment)
- Uses same `/dev/tty` input pattern as `select_project_type()` for non-terminal stdin compatibility
- Respects `TEKHTON_TEST_MODE` for testability
