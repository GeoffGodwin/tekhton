## What Was Implemented

Milestone 0.5: Agent Output Monitoring And Null-Run Detection

- **File-change activity detection in FIFO monitoring loop**: Before killing an agent on activity timeout, the pipeline now checks for file-system changes (via `find -newer` against a timestamp marker) since the last activity check. If files were modified in PROJECT_DIR, the activity timer resets instead of killing the agent. This prevents false activity-timeout kills for JSON output mode agents that produce no FIFO streaming output.

- **Pre-run marker for scoped change detection**: A timestamp marker file (`_prerun_marker`) is created at the start of each `run_agent()` call. All file-change detection is scoped to changes made DURING the agent run, not pre-existing uncommitted changes. This prevents false positives in environments with prior dirty state.

- **Git-based null-run override**: After agent completion or kill, the pipeline checks for file changes since the pre-run marker before declaring a null run. If files were modified during the agent's run, the run is classified as productive regardless of turn count or exit code. This prevents discarding completed work from agents that complete silently (JSON output mode).

- **CODER_SUMMARY.md existence check as completion signal**: If CODER_SUMMARY.md exists, was written during the agent run (newer than pre-run marker), and has 3+ lines, it is treated as evidence of productive work, overriding null-run heuristics.

- **Helper functions added**:
  - `_create_activity_marker()` — creates timestamp marker file
  - `_detect_file_changes()` — fast `find -newer` check excluding .git, session dir, and log dir
  - `_check_git_working_changes()` — authoritative `git status --porcelain` check
  - `_count_git_changed_files()` — estimates scope via `git diff --stat`
  - `_count_changed_files_since()` — counts files changed since marker, excluding pipeline artifacts

- **Backward compatibility**: Text-mode FIFO monitoring continues to work identically to 1.0 behavior. File-change detection is a complementary signal — FIFO output still resets the activity timer as before. The new checks only activate when the FIFO is silent.
