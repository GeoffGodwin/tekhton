### 2. `lib/common.sh` empty-line fallback uses hardcoded width

**Location:** `lib/common.sh:64-65`

`_print_box_line` accepts a `_bw` (box width) parameter. Its empty-separator
branch has a printf-fail fallback:

```bash
echo "${_bv}                                                            ${_bv}"
```

The hardcoded string contains approximately 60 spaces, which happens to match
the default `_BOX_W=60` from `_setup_box_chars`. When called with a non-default
width (e.g., future callers passing 80), the fallback silently renders at the
wrong width.

**Fix:** Add a comment on the fallback line making the assumption explicit:
```bash
echo "${_bv}                                                            ${_bv}"  # ~60 spaces — assumes default _BOX_W=60
```
This is documentation-level, not a code change to logic. Low risk.

### 1. `lib/agent_monitor.sh` exceeds 300-line ceiling

**Location:** `lib/agent_monitor.sh` (332 lines)

The 300-line non-negotiable ceiling is exceeded by 32 lines. One extraction has
already been done (`agent_monitor_helpers.sh`), but the main file remains over
the limit. The Windows/platform detection block at the top of the file (~lines 9-55)
is a cohesive, standalone section: it detects whether the claude binary is a
Windows-native executable and defines `_kill_agent_windows()`. This block has no
dependencies on `_invoke_and_monitor()` or the FIFO reader — it is fully
extractable.

**Fix:** Extract lines ~9-55 (platform detection globals, `_TIMEOUT_KILL_AFTER_FLAG`
setup, `_AGENT_WINDOWS_CLAUDE` detection, and `_kill_agent_windows()`) into a new
`lib/agent_monitor_platform.sh`. Add a `source` (or `.`) call at the top of
`agent_monitor.sh`. Update ARCHITECTURE.md to document the new file. This should
bring `agent_monitor.sh` to approximately 285 lines.

`agent_monitor_helpers.sh` already declares a dependency on `_kill_agent_windows`
from `agent_monitor.sh` — after the split, update that comment to reference
`agent_monitor_platform.sh` as the source of `_kill_agent_windows`.
