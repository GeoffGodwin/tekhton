## Suspected Root Cause Areas

- **Redirect pattern redundancy** — Lines 51, 68 in lib/rescan.sh and line 161 in lib/rescan_helpers.sh use `&>/dev/null 2>&1` (combining `&>` redirect with redundant `2>&1`). Pattern also exists pre-existing in lib/crawler.sh:188, 233. Use `&>/dev/null` alone to clean up.
- **Awk escaping in _replace_section** — lib/rescan_helpers.sh:117 passes `new_body` as awk `-v body=` variable. If section content contains literal backslashes (e.g., Windows paths), awk interprets escape sequences and corrupts the replacement. Use `printf '%s\n'` or heredoc for safer body passing.
- **Early-exit sourcing divergence** — tekhton.sh has three early-exit blocks (--plan, --replan, --rescan) with bespoke sourcing lists that diverge from each other and main pipeline (lines 252+). None source the same set of libraries. Recurring source of "function available in main but not early-exit" bugs (noted as Cycle 1 blocker example).
- **Implicit cross-library dependency** — lib/crawler_deps.sh line 7 header declares dependency on `_extract_json_keys` from detect.sh, but no source-time validation guard. Load-order controlled by tekhton.sh sourcing detect.sh before crawler_deps.sh. Justified for deferral per established codebase convention (no other companion files validate cross-library deps at load time).
