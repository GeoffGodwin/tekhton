## Simple Blockers (jr coder)
- `_do_git_commit()` in `tekhton.sh` (line 1133) calls `_check_gitignore_safety` and `git add -A` redundantly — pre-staging already runs at lines 1028-1029 unconditionally before `generate_commit_message()`. When gitignore issues exist, the user sees the same warnings twice. Remove `_check_gitignore_safety` and `git add -A` from inside `_do_git_commit()` since all staging is already complete by the time it's called.

## Non-Blocking Notes
- `AUTO_COMMIT=true` is set as an unconditional default in `config_defaults.sh:129`. The CLAUDE.md Milestone 15.1 spec says this should be conditional (`true` in milestone mode, `false` otherwise). The HUMAN_NOTES request is satisfied by the current unconditional default, but M15.1 will need to change `config_defaults.sh` to make it conditional. A comment marking this as "to be revisited in M15.1" would prevent future confusion.
- `check_usage_threshold()` in `lib/common.sh:187` uses `grep -oE '[0-9]+(\.[0-9]+)?%' | head -1` to extract the usage percentage. If the `claude usage` output format changes to include multiple percentages (e.g., per-model breakdowns), `head -1` may capture the wrong value. The existing warning-and-allow fallback mitigates this, but a comment explaining which percentage is expected would help future maintainers.
