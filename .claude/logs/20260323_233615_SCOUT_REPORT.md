## Suspected Root Cause Areas

- dashboard.sh line 49 and 74: First-time dashboard init calls `_copy_static_files()` redundantly via both `init_dashboard()` and `sync_dashboard_static_files()`
- diagnose.sh line 114: `_detect_recurring_failures()` called before DIAG_CLASSIFICATION is set, making it a no-op
- diagnose.sh: Missing verdict type filter in `_rule_review_loop()` causal-log fallback allows APPROVED verdicts to trigger branch

- security_rework.prompt.md: Missing delimiters cause prompt injection confusion when `{{SECURITY_FIXABLE_BLOCK}}` is rendered
- health.sh: `emit_dashboard_health()` called before `_hook_health_reassess()` writes pre-reassessment stale data
- health.sh: `assess_project_health()` and `reassess_project_health()` duplicate code paths (lines 70-95 vs 172-196)
- app.js: `trendArrow()` assumes runs[0] is most recent but parser may return oldest-first order
- causality.sh: `trace_effect_chain()` grep on event ID catches false positives in JSON field content
- detect_ci.sh line 176: Dockerfile glob expands to literal string `Dockerfile.*` if no matches exist
- finalize.sh: Hook letter labeling inconsistent (skips 'k'); 14 hooks registered but comments reference fewer
- hooks.sh: `.gitignore` safety check advises `.env *.pem *.key id_rsa .claude/logs/` but not `.claude/dashboard/`
